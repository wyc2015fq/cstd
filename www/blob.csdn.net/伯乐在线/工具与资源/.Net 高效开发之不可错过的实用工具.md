工欲善其事，必先利其器，没有好的工具，怎么能高效的开发出高质量的代码呢？本文为 ASP.NET 开发者介绍一些高效实用的工具，包括 SQL 管理，VS插件，内存管理，诊断工具等，涉及开发过程的各个环节，让开发效率翻倍。
### Visual Studio
- [**Visual Studio Productivity Power tool**](https://visualstudiogallery.msdn.microsoft.com/34ebc6a2-2777-421d-8914-e29c1dfa7f5d): VS 专业版的效率工具。
- [**Web Essentials**](https://visualstudiogallery.msdn.microsoft.com/ee6e6d8c-c837-41fb-886a-6b50ae2d06a2): 提高开发效率，能够有效的帮助开发人员编写CSS， JavaScript, HTML 等代码。
- [**MSVSMON**](https://msdn.microsoft.com/en-us/library/xf8k2h6a.aspx): 远程Debug 监控器 (msvsmon.exe) 是一种轻量级的应用程序，能够远程控制VS来调试程序。在远程调试期间，VS 在调试主机运行，MSVSMON 在远程机器中运行。
- [**WIX toolset**](http://wixtoolset.org/): 可以将XML 源代码文件编译成Windows 安装包。
- [**Code digger**](https://visualstudiogallery.msdn.microsoft.com/fb5badda-4ea3-4314-a723-a1975cbdabb4): Code Digger 是VS 2012/2013 的扩展插件，能够帮助开发人员分析代码。
- [**CodeMaid**](https://visualstudiogallery.msdn.microsoft.com/76293c4d-8c16-4f4a-aee6-21f83a571496): CodeMaid 是一款开源的VS2012/2013/2015 插件，提供代码分析，清理，简化代码的功能。
- [**OzCode**](https://visualstudiogallery.msdn.microsoft.com/36925113-cdce-4953-a5d6-fb3d2912dad7): 非常强大的VS 调试工具。
- [**CodeRush**](https://visualstudiogallery.msdn.microsoft.com/F9CDA5D8-10AF-4CC6-9D17-207222A3FD13): 是VS的提高代码重构和提升效率的VS插件。
- [**T4 Text Template**](https://msdn.microsoft.com/en-us/library/bb126445.aspx):VS中T4 文本模板是生成代码文件最常用的模板文件，这种模板文件是通过编写文本块和控制逻辑来实现的。
- [**Indent Guides**](https://visualstudiogallery.msdn.microsoft.com/e792686d-542b-474a-8c55-630980e72c30):  快速添加缩进行。
- [**PowerShell Tools**](https://visualstudiogallery.msdn.microsoft.com/c9eb3ba8-0c59-4944-9a62-6eee37294597):支持开发和调试PowerShell 脚本和VS2015代码块的工具包。
- [**Visual Studio Code**](https://code.visualstudio.com/): 免费的跨平台编辑器，可以编译和调试现代的Web和云应用。
### ASP.NET
- [**Fiddler**](http://www.telerik.com/fiddler): 能够捕获 http 请求/响应来模拟请求行为。
- [**AutoMapper**](http://automapper.org/): 自动生成对象到对象的映射代码，比如，能够生成从实体对象映射到域对象，而不是手动编写映射代码。Object to object mapping. Like, the tool can be used to map entity objects to domain objects instead of writing manual mapping code.
- **[Unity](https://msdn.microsoft.com/en-us/library/dn170424(v=pandp.30).aspx)/[Ninject](http://www.ninject.org/)/[Castle Windsor](https://www.nuget.org/packages/Castle.Windsor/)/[StructureMap](http://structuremap.github.io/)/[Spring.Net](http://www.springframework.net/)**: 依赖性映射框架，提供很多可用的DI 框架。
- [**.NET Reflector**](https://www.red-gate.com/products/dotnet-development/reflector/): .NET 程序反编译器。
- [**dotPeek**](https://www.jetbrains.com/decompiler/): .NET 程序反编译器。
- [**ILSpy**](http://ilspy.net/): .NET 程序反编译器。
- [**memprofiler**](http://memprofiler.com/): 非常强大的查找内存泄露和优化内存使用的工具。
- [**PostSharp**](https://www.postsharp.net/download): 去除重复编码和避免由于交叉引用产生的代码冗余。
- [**ASPhere**](http://www.asphere.cz/en/index.html): Web.config 图形化编辑器
- [**ComponentOne Studio for ASP.NET**](http://www.gcpowertools.com.cn/products/componentone_studio_asp.htm) 一整套完备的开发工具包
### WCF
- [**SOAP UI**](https://www.soapui.org/): API 测试工具，支持所有标准的协议和技术。
- [**WireShark**](https://www.wireshark.org/):UNIX和Windows系统的网络协议分析器。用于捕获TCP 层的拥塞状况，还能帮你过滤无效信息。
- [**Svc**** TraceViewer**](https://msdn.microsoft.com/en-us/library/ms732023(v=vs.110).aspx): 提供文件追踪视图，是由WFO提供的。
- [**Svc Config Editor**](https://msdn.microsoft.com/en-us/library/ms732009(v=vs.110).aspx): 用于管理WCF相关配置的图形化界面工具。
### MSMQ
- [**QueueExplorer 3.4**](http://www.cogin.com/mq/index.php): 提供消息操作功能，如复制，删除，移动消息，保存和加载，强压测试，浏览编辑等
### LINQ
- **LINQ Pad**: LINQPad 是一个轻量级工具，用来测试Linq查询。 可以测试由不同语言写的.Net 语言脚本。
- [**LINQ Insight**](https://visualstudiogallery.msdn.microsoft.com/269e6da9-7f4d-4650-a327-c70b359dcfe0): LINQ Insight Express 可嵌入 Visual Studio 中，能够分析设计时的LINQ查询 。
### RegEx
- [**RegEx tester**](https://visualstudiogallery.msdn.microsoft.com/16b9d664-d88c-460e-84a5-700ab40ba452):** 正则表达式插件。**
- [**regexr**](http://regexr.com/): 在线正则表达式开发和测试工具。
- [**regexpal**](http://www.regexpal.com/): 在线正则表达式开发和测试工具。
- [**Expresso**](http://www.ultrapico.com/Expresso.htm): 桌面版的正则表达式工具。
- [**RegexMagic**](http://www.regexmagic.com/) : 能够根据文本模式自动生成正则表达式的工具。
### Javascript/JQuery/AngularJS
- [**JSHint**](http://jshint.com/): JavaScript代码质量监控工具，定义了很多非常严格的规则。
- [**JSFiddle**](https://jsfiddle.net/): 提供了浏览器内部的开发环境，能够测试HTML,CSS，Javascript/JQuery代码
- [**Protractor**](http://www.protractortest.org/#/): 端到端的框架，能够测试Angular应用。
### SQL Server
- [**SQL Profiler**](https://msdn.microsoft.com/en-in/library/ff650699.aspx): SQL 跟踪监控工具。
- [**ExpressProfiler**](https://expressprofiler.codeplex.com/): ExpressProfiler (aka SqlExpress Profiler) 是一个小型快速的SQL Server Profiler的替换工具，自带GUI界面。能够用于企业版和非企业版 的SQL Server。
- [**SQL Sentry Plan explorer**](https://www.sqlsentry.com/products/plan-explorer/sql-server-query-view): 提供了SQL 查询执行计划的很好的物理视图。
- [**SQL Complete**](https://visualstudiogallery.msdn.microsoft.com/1e8064cb-58d8-4eb3-a88c-d8f9a167ccca): 为 SQL Server Management Studio and Visual Studio 提供非常智能的，优化SQL 格式的管理工具。
- [**NimbleText**](http://nimbletext.com/):文本操作和代码生成工具。
- [**Query Express**](http://www.albahari.com/queryexpress.aspx): 轻量级的SQL 查询分析器。
- [**IO Meter**](http://www.iometer.org/doc/downloads.html): 提供IO 子系统的一些访问具体情况
- [**sqldecryptor**](https://www.devart.com/dbforge/sql/sqldecryptor/): 可以解密SQL Server 中的加密对象，如存储过程，方法，触发器，视图。
- [**SpatialViewer**](http://spatialviewer.codeplex.com/): 可以预览和创建空间数据。
- [**ClearTrace**](http://www.cleardata.biz/cleartrace/default.aspx): 导入跟踪和分析文件，并显示汇总信息。
- [**Internals Viewer for SQL Server**](http://internalsviewer.codeplex.com/): Internals Viewer 用来在SQL Server 的存储引擎中的查找工具，以及获取数据在物理层是如何分配，组织和存储的。
### NHibernate
- **[NHibernate Mapping Generator](https://nmg.codeplex.com/)**: 生成 NHibernate 映射文件，并从存在的数据库表映射到领域类。
### Tally
**Tally ERP 9**
- [**Tally dll**](http://tallydll.com/): .net 的动态链接库，能够将Tally Accounting 软件集成到应用程序中 ，通过代码对数据进行push或pull操作。
### 代码Review
- **StyleCop**: StyleCop 是静态代码分析工具，能够统一设置代码样式和规范。 可以在Visual Studio 中使用，也可以集成到 MSBuild 项目。
- [**FxCop**](https://fxcopinstaller.codeplex.com/): FxCop 是静态代码分析工具，能够通过分析.Net 程序集保证开发标准。
### 运行状况捕获
- [**WireShark**](https://www.wireshark.org/): It is a network protocol analyzer for Unix and Windows. It can capture traffic at TCP level.
- [**HTTP Monitor**](https://www.charlesproxy.com/documentation/additional/http-monitor/): enables the developer to view all the HTTP traffic between your computer and the Internet. This includes the request data (such as HTTP headers and form GET and POST data) and the response data (including the HTTP headers and body).
### 诊断工具
- [**Glimpse**](http://getglimpse.com/):提供服务器端诊断数据。如 在ASP.NET MVC 项目,可以通过NuGet添加。
### 性能
- [**PerfMon**](https://msdn.microsoft.com/en-us/library/aa645516(v=vs.71).aspx): 使用 性能计数器监控系统性能。

- [**PowerGUI**](http://en.community.dell.com/techcenter/powergui/w/wiki/script-editor#&panel1-1): 能够快接收和使用PowerShell 来有效管理 Windows 开发环境。
