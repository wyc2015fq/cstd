如果你关注Distrowatch网站上最新的发行版资讯，就会发现在过去的12个月，统计数字基本上没什么变化，继续以比较知名、历史非常悠久的发行版为主。

### **2016年值得期待的几大Linux发行版**
让人好生奇怪的，170多个发行版仍处在等待名单上；其中几个发行版的历史甚至可以追溯到五年前；更有意思的是，其中一些发行版实际上受到了相当多人的追捧。这证明了一点：即便某个发行版没有受到 Distrowatch 的认可，它未必就很差劲或没有价值。
虽然 Ubuntu 和 Linux Mint 这些知名发行版永远会在那里，眼下地位可能无法撼动，但绝不可无视那些大有潜力的发行版，知道这一点很重要。
如果你记得 eOS 的故事，这款操作系统在几年前(2011年)几乎流行起来，设法引起了业界的注意，尤其是由于它带来了与主流发行版全然不同的特性。
对许多新的发行版来说，并不是这样子；但是如果你在 Linux 圈子已混迹了一段日子，那么就非常清楚：开发人员推出发行版的速度比过去(比如说十年前左右)大大减慢了。
就如今发布的大多数发行版而言，正是某一项独特的功能/特性让它脱颖而出。下面介绍的那些发行版就是这种情况。
想选出2016年值得期待的发行版特别困难，因为老实说，它们都很出色，只不过一些发行版面向所有用户，另一些发行版提供的功能特性针对一批特定用户，这就是为什么我们会根据需要不断更新这篇文章。
闲话少说，还是赶紧看一看我们选出的2016年值得期待的几个发行版吧！
### **1.VeltOS**
我喜欢把 Linux 归因于这句名言：由社区治理，为社区谋利！你没必要非得赞同我的观点，但是严格上来说，事实确实如此。
VeltOS 由 Velt technologies 开发，这种操作系统生来对 Linux 采取了一种很不同寻常的方法，尤其是在于它会成为一个完全社区驱动的项目。开发人员几乎让你决定想在这款操作系统中看到什么，或者不想看到什么。
![VeltOS桌面](http://ww2.sinaimg.cn/large/0064cTs2jw1ezq2izuy9qj30vj0f6q4v.jpg)
*VeltOS桌面*
![VeltOS 应用程序](http://ww2.sinaimg.cn/large/0064cTs2jw1ezq2j07oyhj30up0exmz4.jpg)
*VeltOS 应用程序*
我喜欢称之为“干脆利落的手腕”，你也可以称之为纯粹的 FOSS。VeltOS 把按自己希望的样子改变操作系统这个权利交给了用户，基本特性及其他一切完全依赖社区。
你几乎可以决定操作系统的外观和行为，感觉就像是在一个投票平台中。你几乎可以发起一项新的民意调查，请求功能特性或者要求删掉某一项功能特性。至于 Velt 最终会是什么样子，完全由你掌控。
然后，有必要知道这一点：VeltOS 还很年轻，他们在三个月前才发布了 TP1(技术预览版1)；它引起了业界的关注，这不足为奇。鉴于这个事实：你可能认为，该发行版提供的并非全新的功能特性(好几个发行版有时实际上采纳用户的建议，帮助改进系统)，VOS 完全旨在由你来构建，符合开源精神。
VOS 很不一样，原因就在于用户就是有远见的人；与社区的其他用户一样，你认为别人需要的功能特性正是你想在操作系统的下一个迭代版本中想要看到的。
VOS 基于 Arch Linux，尤其是由于它具有轻量级、滚动发布的特性；它以用途广泛、使用简单和先进软件尤为出名。Arch 通过 Pacman 软件包管理器提供自己的软件；由于滚动发布模式，你总是会获得最新的安全更新版和软件包。
TP1 实际上就是个测试版本，看看社区在此基础上会把它带到何处。TP1 捆绑了来自 Solus 项目的 Budgie 桌面环境的修改版，还有用于用户会话和欢迎画面(登录屏幕)的 Light 显示管理器以及用于设计美学的 Numix 主题。
快进到2015年12月，VOS 开发人员接到了来自广大用户的无数反馈，了解了 TP2 应该捆绑哪些功能特性。
开发团队目前在努力工作，会使用 Vala 和 C 语言，从头开始编写一个全新的桌面环境，按照 VDE(Velt 桌面环境)的样式。
联合创始人 Stefan Kwiecinski 在接受采访时说：“我们不会分支 Budgie，这方面存在沟通不通的情况。TP1 交付了 Budgie 的修改版。至于 TP2，我们正在努力构建一个全新的桌面环境，用 Vala 和 C 来编写。我们之所以这么做，是由于来自投票平台的选票并不认同 Budgie 现有/未来版本的大部分功能。”
VDE 在开发时会遵循谷歌的 Material 设计原则――实际上牢记谷歌的 Material GUI，会首次出现在 VOS 的 TP2 中。我们联系上了 Stefan，他声称：“如果一切进展顺利的话，我们会在今年3月25日推出TP2。”
这未必意味着我们在今年3月肯定会看到 TP2，有可能提早也有可能延后，因为 VOS 开发团队的规模还相当小。
在此期间，如果你想试一试目前状态的 VeltOS(使用 Budgie 桌面环境的 TP1)，完全可以下载映像文件，不过务必在虚拟机环境下试一下。我们建议使用 VirtualBox 来试一下。
### **2.PapyrOS**
PapyrOS 最先名叫 QuartzOS，后来改叫 QuantumOS，其整个理念完全围绕遵循 Material 设计的操作系统，基于 Arch Linux。
![Papyros应用程序启动器](http://ww3.sinaimg.cn/large/0064cTs2jw1ezq2j0m7fvj311y0lcq3p.jpg)
*Papyros应用程序启动器*
![Papyors网络](http://ww3.sinaimg.cn/large/0064cTs2jw1ezq2j1d1nlj311y0lcjrv.jpg)
*Papyors网络*
Michael Spencer 是项目首席开发员，他最初在2014年年初有了这个想法，后来开发工作在2014年11月迅速开展起来。
PapyrOS 恪守这个理念：操作系统不光适合 FOSS 界的那些人，还应该适合“普通人”。
该项目目前处于初期测试版阶段，Papyros 网站上有一个进度条，随时跟踪开发进展，甚至有一个安卓应用程序会同样给出该项目进展方面的最新信息。
PapyrOS 异常简化，采用了 Material 设计的桌面环境，视觉效果很眩，基于 Arch Linux，旨在将易用性提高到 Linux 界的另一个水平。由于具有 Arch 的滚动发布性质，万一出现了问题，Papyros 会支持原子升级和回滚，这通过 OsTree 来实现。
此外，PapyrOS 将支持与所有常见的 Web 服务集成，包括 Gmail、Inbox 和 Evernote，以改善桌面体验。
抛开上述功能特性外，该操作系统还将随带自己的跨平台应用程序开发框架，该框架将基于 QtQuick。
Papyros 的桌面外壳是用 Qt 5 和 QML 开发而成的(基于 QtCompositor API，目的是为了可以使用 XWayland)，以获得 Material GUI 的高度漂亮、动态的界面，他们一直在将这方面的所有特性实施到操作系统和常规应用程序中。桌面外壳目前可以在基于 Arch 的 Linux 机器上试用。
#### **下载 Papyros Shell 到 Arch 上**
首先，将下面几行代码添加到 /etc/pacman.conf 文件中，在默认的软件库上面：


```
[papyros] 
SigLevel = Never 
Server = http://dash.papyros.io/repos/$repo/$arch
```
然后，运行：


```
# pacman -Syu 
# pacman -S papyros-shell
```
你只要在桌面上的新窗口中运行外壳，就可以对它进行测试：


```
papyros-session
```
进度条目前显示89%，这个值可能有变化，取决于可能包括代码错误发现或请求新功能的新旧情况。
最重要的是，项目在取得进展；等到进度条达到100%，第一个测试版将准备就绪，目前是版本0.1。
其代码是完全开源的，目前放在 GitHub 上。
PapyrOS 承诺拥有的特性绝不仅仅是 Material 主题的用户界面。与 VeltOS 一样，这款操作系统也会针对典型的计算用户，基于 Arch 基础，提供最佳的 Linux 体验。
你可以在 [http://papyros.io](http://papyros.io/) 找到 Papyros。
### **3.Korora**
Korora 已经是一款站稳了脚跟的发行版，最初在2005年时基于 Gentoo Linux，后来该项目在2010年迎来了重生，改而基于Fedora ――实际上就是重新合成，作了一些改动，并添加了其他特性，这直接有助于提高易用性，又不需要系统安装后的后期配置――实际上让你的系统即开即用。
![Korora Cinnamon 桌面](http://ww2.sinaimg.cn/large/0064cTs2jw1ezq2j1spkwj311y0lcjte.jpg)
*Korora Cinnamon 桌面*
Korora 基于 Fedora，实际上不一样，原因在于它提供了第三方软件库，那样你就能轻松获得在 Fedora 上平常无法获得的应用程序，不需要安装系统后的一番后期配置。
Korora 仍与 Fedora 保持着密切的关系，因为它使用后者的默认软件库用于其核心系统更新。该操作系统的主要目的是，大大增加 Fedora 的用户群，同时仍提供出色的技术―― Korora 在这方面一直做得不赖。
据 OzonOS 网页的介绍页面声称，就在最近，Korora 提议将项目与同样基于 Fedora 的 OzonOS 项目合并――OzonOS 不再处于积极开发的状态(实际上由于资金不足，开发团队规模小)，希望以此改进操作 Korora 操作系统的设计美学，那样当然会吸引更多用户，尤其是游戏玩家。
我们很想知道 Korora 开发团队打算如何合并 OzonOS 与其项目。到目前为此，还没有具体实施；但如果我没猜错的话，2016年应该会看到 Korora 方面的一些变化。
Korora 目前有几种不同的版本，即 Gnome、KDE、XFCE 和 Mate，它们几乎有着同样的基础。
另外值得一提的是，Korora 会随带支持第三方驱动程序的功能，支持所有的英伟达和 AMD 图形卡用户。
正如你所预料，该项目是开源项目，你可以在其[官方网站](https://kororaproject.org/)及 GitHub 帐户上找到更多的信息。
### **4.SolusOS**
我们当然知道 Debian、Arch、Ubuntu、Gentoo 以及各自的衍生版，它们林林总总共有300多个不同的发行版，其中大多数适合特定的受众，有几个脱颖而出。
你目前使用的发行版很有可能基于上述这些母体版本，不过你觉得全新、全然不同的发行版怎么样?
![SolusOS 桌面](http://ww3.sinaimg.cn/large/0064cTs2jw1ezq2j28o2dj30sg0g0n1w.jpg)
*SolusOS 桌面*
![SolusOS Steam](http://ww1.sinaimg.cn/large/0064cTs2jw1ezq2j3gdxsj30sg0g0aad.jpg)
*SolusOS Steam*
Solus 项目最初是以 SolusOS 开始的，基于 Debian，此后开发人员把它划掉，开发出了第二个 SolusOS (Solus OS 2)，这显然是所有当前工作(直到全新的 SOS)的前身，它是从头开始构建的，之后该项目被关闭。
快进到2014年2月，EvolveOS 问世，几乎共享与 SolusOS 的第二个版本(已关闭)同样的代码库和 Vision。
EvolveOS 看到早期的 Alpha 测试版变成了 Beta 版本，之后该项目的名称遇到了一点问题，后来重新改为 Solus Operating System――实际上就是第一个版本，只不过 SOS 完全归属 Solus 项目，“包括项目负责/维护的所有工作(因而缓解了法律问题)”，Ikey 如是说。
再来说说细节方面！Ikey Doherty (项目创始人兼管理人)开始 SOS 时，主要目的是从头开始构建一款操作系统。
SOS 有一个全新的桌面环境：基于 Gnome 3外壳的 Budgie；实际上，该桌面环境充分利用了底层的 Gnome 3堆栈，同时提供了自定义窗口管理器：Budgie-wm，以及名为 Budgie-panel 的面板。这个面板展示了新的侧边栏，并对用户界面作了其他诱人的大量改进。
据 Ikey Doherty 声称，界面已经过重写。
SOS 使用 PiSi 软件包管理器和 CLI 中的 eopkg。当然，eopkg 命令不像你通常所熟悉的 apt-get 命令和 pacman 命令。然而，你可以在 apt-get 和 PiSi 之间找到首席开发员 Ikey Doherty 发布的对应命令。
为了强调这一点，我们联系上了首席开发员 Ikey，据他声称“软件包格式是 .eopkg，软件包管理器大约3年前从 Pardus Linux “PiSi” 软件包管理器(不是 Pisi Linux 项目)分支出来。”
该发行版在12月25日迎来版本1.0，在2016年肯定会看到 SOS 出现重大变化和改进。
你可以上[官方网站](https://solus-project.com/)了解更多信息。
