# 普天同庆：GitHub提供免费私有Repo，真的贼好用啊～ - 知乎
# 



> 新的一年，GitHub 开放免费的私有 Repo 啦，还是无限量的～

以前 GitHub 的所有私有项目都需要收费，因此尽管开源项目大多都集聚在 GitHub 上，企业和个人的私有项目都放在了其它代码托管平台，例如国内的 Coding 和国外的 GitLab 等。现在，这一免费的私有项目策略，正是微软收购 GitHub 后的首次重大变革。

具体而言，GitHub 宣布两大主要更新：无限免费的私有代码库和更简单统一的企业产品。这种定价策略的改变，至少对个人开发者有非常大的影响，我们可以节省很多使用成本。

**个人与团队两大更新**

GitHub Free 现在包括无限的私有代码库。首先，开发者可以在 GitHub 上存储私人项目，每个库最多有三名合作者。很多开发者想在公开发布私有代码库之前，将其应用于求职、工作、辅助项目或者私下尝试其它东西。从今天开始，GitHub 免费提供这些服务以及更多其它的服务。开源代码库仍是免费的，合作者人数不限。

GitHub Enterprise 是 Enterprise Cloud（正式名称为 GitHub Business Cloud）和 Enterprise Server（正式名称为 GitHub Enterprise）的结合产品。想要在云端或 self-hosted 配置中灵活使用 GitHub 的组织现在可以以一个的价格获取这两个功能。此外，这些产品可以通过 GitHub Connect 得到安全连接，从而提供一个混合选项，使开发者在两个环境中无缝切换。
![](https://pic3.zhimg.com/v2-6f4c54e5d60388b09d7c2a6a1da85da2_b.jpg)![](data:image/svg+xml;utf8,<svg xmlns='http://www.w3.org/2000/svg' width='788' height='565'></svg>)对于个人开发者而言，Free 版和 Pro 版功能上的区别在于私人 repo 合作者数量（3 VS. unlimited），以及 Pro 版用户可使用高级代码 review 工具。![](https://pic1.zhimg.com/v2-74bccb13064c48e8f41405ba9625962c_b.jpg)![](data:image/svg+xml;utf8,<svg xmlns='http://www.w3.org/2000/svg' width='716' height='821'></svg>)GitHub 新的企业和团队定价，其中开源团队仍然是免费的，只不过启用私有项目的团队和企业需要付费。
需要专业编码和合作功能的开发者和团队还可以使用 GitHub Pro（正式名称为 GitHub Developer）和 GitHub Team。当然，开源贡献者仍将拥有他们合作公共 repo 所需的所有功能，包括免费版的 GitHub Team。

无论你是即将写第一行代码的学生，还是团队遍天下的企业领导，抑或开源维护者，GitHub 团队都希望 GitHub 能够成为大家编程、合作、与全球开发者社区连接的最佳区域。今天发布的这一变化是对 GitHub 未来的一次巨大投资！

而对于广大用户和开发者来说，这次开源更是一次大大的福利。来看看他们怎么说↓↓↓

**太棒了，可以省钱了！**

reddit 网友 ibrentlam 表示：赶紧把账户降级成免费版了，每个月可以省 7 美元！哦耶！

hacknews 网友 mark_l_watson 表示：这可真是一个大惊喜。我有很多公开的 GitHub 项目（我写的各种书的代码以及一些小型开源项目）。因为我喜欢把这些粗糙的代码实验保密，所以也为了 Pro 用户付费了多年。我接受了微软/GitHub 的新计划，并在年度账单结算时取消了 Pro 计划。感谢微软爸爸！

**天下没有免费的午餐？**

而 reddit 网友 alitterbox 则表示：GitHub 是想获得尽可能多的源代码，我确定他们在尽量挖掘去发现这些源代码有什么。没有什么会是「免费」的。

网友 pengo 持相反观点：

> GitHub 毕竟现在属于大公司，就像其他云服务一样。使用时应该像使用其他云服务一样谨慎，因为云提供商也是可以看到你的数据的。

不过，这仍然是一项免费服务，所提供的服务和之前付费提供的服务是一样的，这与挖掘信息无关。不过一旦你的项目有超过三个人，那么 GitHub 将会收费。这是微软/GitHub 一项明智的商业计划。之前，你可能需要使用另一项免费服务（如 BitBucket）来开始构建你的私有项目，因此微软选择扩展免费等级并不令人吃惊。

用不着阴谋论。

Hack News 上有网友表示：

> 2016 年 GitHub 财务状况披露的时候，其个人账号所得收入占到了总收入的 12.5%，GitHub Enterprise 则占 50%，剩下的 37.5% 来自组织（企业）。公平地说，从收入的角度来看，个人账户的收入无关紧要的。

12.5% 的个人账户收入无关紧要？有网友站出来反驳，「如果你的收入减掉 12% 你也觉得无所谓，那我无话可说。」

但也有网友觉得牺牲掉这 12.5% 很值：

> 你只看到了短期影响。我大学的时候就开始用 GitHub，我所有的项目都在上面，我的队友们也是如此，GitHub 真的很好用。但当我的学生账号到期后，我就决定将所有的东西都搬到 GitLab 上。我并不想这么做，但我也不想每个月付 7 美元去保留一些上学时的旧项目以及一些私人文件。移到 Gitlab 之后，我觉得这个平台也挺好用的。如果我要开始一个商业项目，你觉得我会选择哪个平台？过去我可能会选择 GitHub，但现在我习惯了 GitLab，也看到了它所有的优点，我可能会选择 GitLab。去掉那 12% 可能让你在短期内损失 12%，但几年之后，你得到的远比那 12% 多得多。
