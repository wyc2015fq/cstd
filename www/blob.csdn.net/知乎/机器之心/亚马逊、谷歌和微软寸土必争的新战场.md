# 亚马逊、谷歌和微软寸土必争的新战场 - 知乎
#

> 当亚马逊、谷歌与微软都表示要争取所有可能的垂直市场客户时，云服务三巨头在零售市场的竞争变得愈发有趣。

**作者****| Caroline Donnelly**

**编译****| 张玺**

**云技术在零售市场应用方面表现抢眼，但是竞争问题正在影响客户的购买决策。**

当亚马逊、谷歌与微软都表示要争取所有可能的垂直市场客户时，云服务三巨头在零售市场的竞争变得愈发有趣。

三巨头的参考客户名单中有妇孺皆知的超大型零售商，这些零售商试图充分运用云技术改善其线上与门店客户体验。

英国电商网站 Ocado、英国化妆品 Lush Cosmetics、Target 与 PayPal 等零售品牌的 IT 基础设施均由谷歌云平台（GCP）托管 ；而微软 Azure 的公开客户则包含 Marks & Spencer（玛莎百货）、英国零售品牌 ASOS 与 DixonsCarphone。

亚马逊网络服务（AWS）吸引零售客户方面同样表现不俗──英国老牌商超 Sainsbury's（英佰瑞）与便利店品牌 Nisa 都使用 AWS 技术，以及诸多线上零售商品牌 Shop Direct、高街时尚品牌 River Island 与葡萄酒超市 Majestic Wine。

当然，与亚马逊线上商城比起来，这些就显得相形见绌了。亚马逊线上商城不仅是 AWS 在册最大零售客户，还是全平台最大的客户。

但对于某些零售商来说，亚马逊公司与 AWS 关系实在是过于紧密，以至于它们无法放心地将自己公司的应用程序与工作负载委托给这家云服务巨头。

## **沃尔玛的反抗**

2017 年 6 月，美国零售巨头沃尔玛发明声明，澄清「华尔街日报」的一篇报道，报道指出沃尔玛警告其技术合作商从 AWS 迁移出托管的应用程序、工作负载与数据。

沃尔玛对此回应，「我们的供应商拥有选择任何满足其自身与客户需求能力的云服务提供商的权利。针对我方倾向于将敏感数据迁移出竞争对手平台的情况，大家不应太过意外。」

不过，这个回应不久就因为当事人——亚马逊的声明而「打脸」，亚马逊表示前者曾试图以「霸凌」方式要求供应商放弃 AWS 技术，而理由也很「莫须有」──亚马逊的云服务事业部会「以某种方式帮助零售事业部」。

在一来一回的「口水战」之后，沃尔玛与 AWS 的竞争对手微软签订了一项为期数年的云服务协议，还被开源私有云平台 OpenStack 列为参考客户。

## **当客户利益面临冲突**

最近，AWS 参考客户、新加坡零售商 Trax Retail 宣布增加公司云服务提供商数量，其中就包括谷歌云（Google Cloud Platform，以下简称 GCP），从而将客户之间的利益冲突问题摊到台面上来。

Trax Retail 宣布将开发数据分析软件，辅助零售商设计门店布局，利用门店移动终端采集的图像数据管理库存水平。

本月初，2019 年度的零售业联合会（NRF）大会在纽约举行。Trax Retail 美国管理负责人 David Gottlieb 在大会上接受了「计算机周刊（Computer Weekly）」的采访，「在零售行业存在一种很奇特的情况，部分零售商不愿意使用亚马逊公司建立的技术生态系统。因为他们认为会与亚马逊公司在门店销售方面存在竞争关系。」

Trax Retail 在大会上首次公开发售 Retail Watch 方案，通过在门店货架上安装大量定制化设计的物联网摄像头，帮助零售商监控所有物品的库存水平。

Trax Retail 强调，采用 Retail Watch 能使零售商了解库存实时变化情况，其数据处理业务交由谷歌云完成。

Gottlieb 说，「我们有特许经营权，能直接与大众消费品制造商协作。过去，所有系统部署都在 AWS 上。目前，我们仍为那些制造商提供 AWS 程序部署选项，但我们想寻找新的合作伙伴。与 GCP 的合作对我们很重要。」

## **多服务商，还是单一托管？**

CCS Insight 企业研究副总监 Nicholas McQuire 认为，采取多云服务商的策略本身也是解决客户利益问题的一种方法，但却不适用于所有人。

**大多数零售商仍处于数字化转型初级阶段，在面对许多不同云服务提供商时，只能碰运气。但从长远来看，这种方法不实用且不经济。**

McQuire 补充道，无论 AWS 提供的优势是否超过其他可选择潜在竞争对手，零售商发现在某些情况下必须加大投入。

McQuire 说：「不只是零售组织，许多公司都采用多云服务商的策略，但如果真的想完全打乱市场、重构组织，他们可能必须要判断多云服务商策略是否持续可行。这可能会提醒某些公司挑选出一家首选云服务提供商，并与其深入合作。因为与一家提供商深入合作会产生更多的经济利益与发明创新。」

最近，就有一个类似的例子。云端电子平台提供商 BigCommerce 将超过 6 万家使用其服务进行线上交易的零售商，从 IBM 的 Softlayer 与 AWS 转移到 GCP。

**「计算机周刊」曾报道过，BigCommerce 过去依赖 IBM 托管基础设施群，而在高峰访问周期内运用 AWS 云技术处理访问量，但现在却签约谷歌以解决延迟及性能问题。**

「迁移至 GCP 的决策主要是受我们客户的驱使，兼顾了他们及他们客户的位置。」BigCommerce 在「计算机周刊」的声明中说道。

**「谷歌是唯一家自行铺设的全球光纤与海底线缆网络的托管提供商。这非常值得注意，因为它让所有商户尽可能快速的与客户进行交易。我们认为谷歌在网络方面的基础设施具备极强的竞争优势。」**

## **微软的新零售算盘**

在 2018 年的美国零售业联盟展览会上，微软公布了和药企沃博联（Walgreens Boots Alliance，WBA）一项为期七年的云服务协议。在美国及欧洲，包括英国高街连锁 Boots，沃博联旗下有许多知名的医药及化妆品品牌。

2014 年，由美国老牌药店零售商沃尔格林和英国药妆连锁品牌联合博姿两家公司合并，沃博联正式诞生。目前，沃博联的全球雇员超过 41.5 万人，且承诺将「大多数」IT 基础设施迁移至微软 Azure 共有云，且公司 38 万员工全面使用微软 365 线上软件包。

沃博联在公司的新闻声明中表示，此举的首要目标是帮助改变医疗服务的送达方式，以使其面向客户更易于使用、人性化且能够负担。IT 市场观察者盖特纳（Gartner）的研究副总监、著名分析师 Ed Anderson 向「计算机周刊」表示此协议的重要性不容小觑，因为有许多用户选择此种方式，而不是直接上云或迁入共有云。

Anderson 说「Azure 与 Microsoft 365 赢得了一个大客户。同时，这也是一个长期承诺──与使用微软技术的零售与医疗保健行业的领导者共同创新。同时，还表明微软技术的未来发展能够提供数字化转型解决方案。尽管还不能确定这个长期计划会发展成什么样，但现有细节已足够凸显，该笔交易未来一定会产生的一些有趣的联合解决方案。」

与此同时，零售商亚马逊也在其旗下医疗保健零售部门一起推进，包括收购线上药房 PillPack。

一年前，沃博联与金融服务巨头摩根大通及保险公司伯克希尔·哈撒韦联合投资，改善这些公司向美国国内雇员提供医疗服务的质量与经济性。

当亚马逊开始逐渐占领沃博联的市场时，大家肯定想知道这对于沃博联全面选择微软云服务的决策是否有影响。

盖特纳的 Anderson 说：「毫无疑问，许多零售商都在寻找 AWS 之外的技术提供商，因为他们感受到竞争冲突。这使得某些零售机构选择微软、谷歌及其他云提供商。然而，AWS 同时也吸引了许多零售商。所以，现在还不能断言 AWS 在零售市场一定会失败。」

盖特纳也通过公司发言人向「计算机周刊」表明此观点，「在大多数亚马逊零售友商中，还是非常流行使用 AWS」。

AWS 发言人说：「许多零售商之所以继续选择 AWS 是因为他们认为大家需要的技术基础设施平台所提供的功能、敏捷、安全与性能，都应该是顶级的。而 AWS 在这些方面是毫无争议的领先者。零售商用户不关心可能存在其他零售商对手。在快速发展的时代，终端用户只关系客户体验的迭代速度是否足够快以确保他们与零售商交易。速度与能力真的很重要。」

## **亚马逊要和竞争对手做朋友**

实际上，有许多与 AWS 存在直接竞争关系的客户公司仍在使用其云服务的例子。比如，流媒体服务商 Netflix 是 AWS 的长期稳定客户，其提供的核心产品直接对标 Amazon Prime Video 的产品。

在这点上，还有件事值得注意注。2019 年 1 月初，AWS 在上议院通信委员会召开的互联网监管听证会上也强调过：为了能在亚马逊商城上销售他们的产品，多数零售商不顾自身竞争优势。

基于亚马逊英国及爱尔兰地区公共策略负责人 Lesley Smith 在听证会上公布的数据，亚马逊商城中超过五成的销售物品由第三封零售商提供，包括散户商家及旗舰品牌。

Smith 向委员会阐述，亚马逊商城的目标是保证客户能在线买到「任何所需所想」，而要想完成此目标就需要与这些品牌合作，不是竞争。

Smith 说：「我们想变得更有优势，吸引更多客户，但是挤走友商来说对我们没有任何好处。这点我从未迟疑。我们现在也不是这么干的。」

此外，Smith 还分享了一些案例，亚马逊是如何与帮助零售商在英国高街提高销售额──通过在本地邮局及合作商超引入亚马逊商品的点击提货（Click and Collect）门店。

Smith 补充道：「人们不会只宅在家里刷着亚马逊。他们有许多选择。」实际上，许多零售商愿意与亚马逊商城合作销售产品或为 Click and Collect 门店提供场所，**是因为他们相信亚马逊与他们利益趋同。**对于 AWS 来说，今年的关注焦点应该放在与合作的零售商建立平等的信任关系。

CCS Insight 的 McQuire 说「展望 2019，AWS 的重要工作是建立市场信任，尤其是亚马逊零售机构已经搅乱的那些垂直行业。随着企业在未来数字化转型与服务云化的深入，作为亚马逊集团内部实体的 AWS 如何才能 [成为] 企业值得信任的平台。」

## **监管质询数据来源**

亚马逊与其他巨头技术公司已经在美国与欧洲监管当局的严密审查下，找到了适合自己的工作模式。监管者之前过度关注技术巨头的规模及影响是否会在其运营市场形成独占垄断。

该议题在上议院通讯委员与亚马逊进行的高层会议上也讨论过。当时，Smith 严辞决绝，并明确表示亚马逊在云服务与线上零售并非处于支配地位。

此后，欧盟竞争委员会确认了关于亚马逊零售平台如何使用商户数据的初步调查，该调查开始于 2018 年 9 月。

在上议院听证会质询此问题时，Smith 反对关于亚马逊商城能够使用在任何在其平台上销售产品的第三方商户的销售数据的指控。

Smith 说：「我们没有拥有此权限。市场销售者能查阅商品如何销售，但是我们的营销人员无法看到销售数据或者关于市场销售者的任何关联数据。」

尽管已作出类似声明，听证会上反复提及一套针对如何解决亚马逊影响及数据使用问题的解决方案──拆分亚马逊。

但是，McQuire 不相信此举会明显改变零售商关于 AWS 与亚马逊商城之间关系的看法。即便未来发生拆分，监管当局也不应该参与其中。

McQuire 说：「实际上 AWS 与亚马逊零售业务的独立性远超大多数人的想象，但是亚马逊太过看重 AWS 这颗摇钱树，所以即便未来拆分也会是内部行为。针对未来是否会拆分，更多是看政府如何应对此问题，这也是欧盟裁决异常关键的原因。」
