# 彭博确认微软即将收购GitHub：喜提最大代码托管网站 - 知乎
# 



选自彭博社，**作者：Dina Bass、Eric Newcomer，机器之心编译。**

> 在上周五传出消息后不久，微软对GitHub的收购邀约很快成为了现实。据彭博社等媒体得到的消息：微软已于上周日决定收购这家最热门的开源代码共享平台。正式的收购通告最早将在今天发布。
![](https://pic1.zhimg.com/v2-0f2915c798a103995d4396fb22ef4fd8_b.jpg)![](data:image/svg+xml;utf8,<svg xmlns='http://www.w3.org/2000/svg' width='1080' height='719'></svg>)
6 月 3 日得到的消息，微软已决定收购开源代码项目托管平台 GitHub，收购金额据称为 50 亿美元。与此同时，原微软高管 Nat Friedman 或将成为 GitHub 的新任 CEO，若最终成行，他将成为自去年 8 月联合创始人 Chris Wanstrath 卸任之后公司的第一位新 CEO。此前，据 Business Insider 等多家媒体的报道，GitHub 一直在与微软等公司进行有关收购的谈判。GitHub 一直倾向于将自己出售给一家上市公司，而微软首席执行官 Satya Nadella 对于 GitHub 印象深刻成为了本次交易的契机。

GitHub 的上一次融资还在 2015 年 7 月。当时，红杉资本等机构领投了 GitHub 价值 2.5 亿美元的 B 轮融资，公司估值为 20 亿美元。

这次收购可为 GitHub 找到新的前进方向：由于 10 个月前 Chris Wanstrath 的交权，这家公司一直在寻找新的领导者（在此期间，GitHub 的首席商务官 Julio Avalos 加入了公司董事会，接管了日常领导工作）。同时，虽然 GitHub 提供的代码托管服务受到了全球用户的欢迎，但公司仍未从主营业务中获得利润。

众所周知，GitHub 是程序员进行代码管理的重要工具。微软、谷歌、腾讯等科技巨头一直在使用 GitHub 对自己的项目代码进行存储与协作。GitHub 也是开发者的社交平台，每天都有越来越多的开发者把自己的项目上传到这里展示和分享。GitHub 的火热为公司带来了很高的运营成本，在 2016 年第三季度，该公司报出亏损 6600 万美元——而在该年度 9 个月中，公司的收入仅达到 9800 万美元。

作为著名代码托管网站，GitHub 已有 2,700 万名注册用户，代码库的数量则超过 8000 万个。

目前，GitLab 已在其官网上发布了一篇文章表示对于此次收购的祝贺。
