# 没有问题约等于没有认识                           

已有 2767 次阅读2012-5-28 09:00|个人分类:[随想](http://blog.sciencenet.cn/home.php?mod=space&uid=279992&do=blog&classid=112648&view=me)|系统分类:[科研笔记](http://blog.sciencenet.cn/home.php?mod=space&do=blog&view=all&uid=279992&catid=1)|关键词:office,normal|[normal](misc.php?mod=tag&id=2611), [office](misc.php?mod=tag&id=2821)





看《数论中未解决的问题》（*Unsolved Problems in Number Theory*），想起老爱的一句俗话：提出一个问题往往比解决一个问题更重要——在其他场合他还说过更“定量”的话：We need to first define the problem. If I had an hour to save the world, I would spend 59 minutes defining the problem and 1 minute finding solutions.


这几天为一个课题设计题目，更体会了这句话的重要。我发现，我能提出很好（自以为）问题的，都是我有很多“感悟”的，还没做就知道结果大概是什么样子的；而提不出具体问题的，是我也很模糊的，不知道结果会怎样的。我对一个同学说，提出的问题都好做，提不出问题是我自己无知。



应该提什么问题呢？老爱说，“解决一个问题也许仅是一个数学或实验上的技能”，那么问题就是为实验或数学提供舞台。很多时候我们搭不起那样的舞台，而只能反复问一个个抽象的“为什么”，那就只好做愚公，一代代问同样的问题，做同样的事情。
