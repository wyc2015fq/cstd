# [完整的直播+点播全端解决方案](https://www.cnblogs.com/dhcn/p/7130753.html)
1、直播端服务器：可以采用中国人的开源作品：Simple RTMP Server，别看名字叫simple，其实一点都不simple，功能完盖大多数第三方直播服务商。
Simple-RTMP-Server 服务器搭建:http://www.cnblogs.com/meetrice/p/5428975.html
https://github.com/ossrs/srs/wiki/v1_CN_LinuxService
2、直播移动端：推荐kslive，金山直播云对外开放的移动端SDK:
https://github.com/ksvc/KSYLive_Android
https://github.com/ksvc/KSYLive_iOSa
3、直播Web端：上面的Simple RTMP Server提供了建议解决方案:
http://www.ossrs[.NET](http://lib.csdn.net/base/dotnet)/srs.release/trunk/research/players/srs_player.html
4、点播服务器：采取比较简单的Nginx http flv.
Ubuntu 14.10下基于Nginx搭建mp4/flv流媒体服务器(可随意拖动)并支持RTMP/HLS协议(含转码工具): http://www.cnblogs.com/jerrychen/p/4705019.html
5、点播移动端：也可以采用kslive的播放器
6、点播Web端：上面的Simple RTMP Server提供了建议解决方案
7、直播CDN推荐，支持RTMP峰值收费最便宜的CDN：星域CDN。支持RTMP且可按量收费的CDN：又拍。
8、点播CDN，如果采用http flv的话，当然是你用哪家的服务器就用那家的http CDN
