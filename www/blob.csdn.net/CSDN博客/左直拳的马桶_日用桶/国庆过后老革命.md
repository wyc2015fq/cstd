老革命永远会遇上新问题。我就是一个老革命。
今天部署系统到客户的服务器。小菜一碟。步骤如下： 
1、装oracle12C 
2、建可插拔数据库，表空间，用户，导数据 
3、部署CS系统 
4、部署WEB项目
第1步由同事完成了。
2 建可插拔数据库还算顺利。但建好后，连接之前，记得要修改tnsnames.ora文件，否则无法解释连接串。因为脚本建的话，是不会自动修改该文件的。
然后导好数据之后，服务器连没问题，但其他机器就不行，说没有监听。
后来找到listener.ora。它只监听了localhost。赶紧改改：
```
LISTENER =
  (DESCRIPTION_LIST =
    (DESCRIPTION =
      (ADDRESS = (PROTOCOL = IPC)(KEY = EXTPROC1))
    )
    (DESCRIPTION =
      (ADDRESS = (PROTOCOL = TCP)(HOST = localhost)(PORT = 1521))
    )
    (DESCRIPTION =
      (ADDRESS = (PROTOCOL = TCP)(HOST = chenqu2)(PORT = 1521))
    )
    (DESCRIPTION =
      (ADDRESS = (PROTOCOL = TCP)(HOST = 192.168.230.1)(PORT = 1521))
    )
  )
```
重启listen服务后，稍等片刻，问题解决。
3、部署CS系统，说没.NET 4.*，装就是了，小菜一碟。但由于IIS安装在前，装好net framework后，要注册一下。找到那个aspnet_regiis，在DOS窗口一运行，说要administrator才行。靠，真变态，我的账户属于administrators组好不好。在windows\system32\下找到cmd.exe，然后用”管理员身份运行“，注册成功。
4、部署WEB。一访问，提示oracle client缺乏之类的错。一怒之下重启，再访问，问题解决。IIS就是贱，逼我出大招。
