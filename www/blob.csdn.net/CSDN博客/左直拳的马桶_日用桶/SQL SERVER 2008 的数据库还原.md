# SQL SERVER 2008 的数据库还原 - 左直拳的马桶_日用桶 - CSDN博客

SQL SERVER 2008的数据库备份，在SQL SERVER 2005下是不能还原的，只能在2008下还原。
如果待还原的数据库，在实例中没有（比如，你想将A服务器上的数据库备份，拿到B服务器上还原一个），那么，不要新建一个数据库，然后指定还原到这个新数据库上。这样是还原不了的，提示什么数据库不相同。
正确的做法是：点实例，任务，选还原，选好备份文件，然后在待还原的数据库下拉框上输入数据库名称（可以输入的，尽管那是一个下拉框），看看选项，里面有各种文件的路径。这个路径，应该跟源数据库的路径一样的。可以修改，但如果文件很多，就很麻烦。我通常是建立一个跟源数据库一样的路径。
这样就可以还原鸟。
我记得2005里面，采用新建一个异名数据库，然后还原是可以的，但不知道为什么2008下不行。
