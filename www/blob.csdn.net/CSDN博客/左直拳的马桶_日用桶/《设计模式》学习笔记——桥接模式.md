桥接模式的定义是将抽象部分与实现部分分离，使它们都可以独立地变化。
这是什么意思？抽象与实现不是本来就分离的吗？比如父类是抽象类，子类是具体实现类。
其实在桥接模式里，主要是用于处理2个或以上维度会变化的情况。比如画笔有大、中、小三种，然后可以蘸上各种颜色进行画画，那么存在两个维度：画笔大小，颜色。用桥接模式来处理，抽象类处理一个维度，而实现类处理另一个维度，抽象类与实现类不是继承关系，而是关联关系。桥接模式用合成复用代替了继承。所谓的“桥”，就是指关联。
![](https://img-blog.csdn.net/20170316063726704?watermark/2/text/aHR0cDovL2Jsb2cuY3Nkbi5uZXQvbGVmdGZpc3Q=/font/5a6L5L2T/fontsize/400/fill/I0JBQkFCMA==/dissolve/70/gravity/Center)
我估计，如果有多个维度，那么抽象类仍然只处理其中一个维度，而每个维度都有一个实现类，然后关联到抽象类。
说到关联，桥接模式和装饰模式何其相似！
如果硬是要说区别，
桥接模式：
```java
class Client {  
    public static void main(String args[]) {  
        Image image;//对应抽象类  
        ImageImp imp;
        image = (Image)XMLUtil.getBean("image");  
        imp = (ImageImp)XMLUtil.getBean("os");  
        image.setImageImp(imp);  //选用具体的实现类
        image.parseFile("小龙女");  
    }  
}
```
装饰模式
```
class Client
{
       public  static void main(String args[])
       {
              Component  component,componentSB,componentBB; //全部使用抽象构件定义
              component = new Window();
              componentSB = new  ScrollBarDecorator(component);
              componentBB = new  BlackBorderDecorator(componentSB); //将装饰了一次之后的对象继续注入到另一个装饰类中，进行第二次装饰
              componentBB.display();
       }
}
```
多轮装饰之后，返回来的还是那个对象。要不怎么叫装饰呢，墙壁粉刷多少遍，它都是那堵墙。你大妈早已不是你大妈，但你大爷始终都是你大爷。
由以上比较可知，
桥接模式由主体对象（抽象类）聚合各种具体实现类，从而扩展了功能，有点建造者模式的味道；实现方式为聚合。
装饰模式则为主体对象穿上层层马甲，改头换面；装饰一遍，扩展一次功能。实现方式为注入。
