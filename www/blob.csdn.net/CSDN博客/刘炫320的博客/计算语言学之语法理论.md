语法理论主要介绍关于语法的相关理论，具体的，主要要介绍一下功能合一文法以及合一运算和代真过程。并对其他语法再做简要介绍。
# 1. 语法理论起源
20世纪50年代乔姆斯基提出4类形式文法（之前讲过）后，他又在20世纪80年代时，提出了管辖与约束理论。后来研究者又提出了词汇功能语法、广义结构语法、树连接语法以及链语法等。
语法理论主要是为了描写自然语言的结构与语义两个方面，另外，也是想揭示人类语言的共性和人类思维的奥秘。因为之前一直以为，语言能力是人类与生俱来的能力。
# 2. 管辖与约束理论
管辖与约束理论认为，语法由两大系统构成：1、规则系统；2、原则系统。
规则系统包括词汇规则、语类和转换规则、语音规则和语义规则等。而原则系统则包括X理论、格理论、管辖理论等等。
这里提一下$\theta$理论。$\theta$理论属于语义范畴，研究词语所处的句法位置与它们在句子描写的动作中所起的作用关系。它试图揭示句子中陈述的行为、各个行为参与者的性质和相互关系，以及这些参与者在句子位置中应受何种约束。
这里似乎没有什么值得说明的地方，但是参与者的角色里就有意思了：参与者的角色包括施事者、受事者或主题、行为的地点、主题的来源、目的地等。这些角色通称为题元（$\theta$-role）。 $\theta$-理论规定：在句子中，每个题元必须以名词的形式出现，且只能出现一次。
但是既然我们这里叫做管辖与约束理论，这两个理论必须要介绍。管辖理论指的是从结构上定义哪些成分可以管辖，哪些成分可以把格赋给另外一些成分。可以赋格的成分称为主管成分，如：动词，介词等；受管辖的成分称为受管成分，如：名词，形容词等；这种关系称为管辖关系。管辖理论研究成为主管、受管成分的条件；研究代词、空语类以及它们与先行词之间的管辖关系等。
而约束理论则是研究名词词组之间的指称关系，在怎样的结构范围(辖域)内，代词(你、我、他、它等)、反身代词(自己等)、指称语、空语类等，可以受到先行词的约束，与先行词同指。
例如：在水边的草地上，一头牛啃着嫩草，放牛娃背对着它在吹笛子。这里的它指的是牛、嫩草还是放牛娃？
总而言之，管辖理论、格理论讨论管辖范围、管辖对象；格理论、θ－理论则在结构和语义两个层次上讨论同一个对象；约束理论和控制理论都是研究名词性词组之间的同指问题；几种理论相互补充、相互交互、相互限制，以解释各种语言现象。
# 3. 功能合一文法
功能合一文法是1985年斯坦福的Martin Kay提出的。它主要针对乔姆斯基的短语结构语法生成能力太强，以致于产生许多不符合语法或者有歧义的句子。
功能合一文法是对短语结构语法的改进：
- 采用复杂特征集来描述词、句法规则、语义信息，以及句子的结构功能。
- 试图以单一形式的结构模式来描述特征组合、功能分配、词条和组成成分的顺序，以达到对句子的完全功能描述。
- 采用合一运算对复杂特征集进行运算。
这里我们介绍一下复杂特征集的功能描述，其实就是一个类似Json格式的描述形式，例如下面这种例子。 
$\begin{bmatrix}Cat=N         \\Sen=Equipment \\Let=计算机\end{bmatrix}$
这就是一个描述计算机的复杂特征集，包括了词性（Cat）、类型（Sen）和词（Lex）。当然你想描述成什么样子取决于自定义的。再比如： 
$\begin{bmatrix}Cat=Verb         \\Tense=Past\\Transitivity=Mental-Process \\Root=see\\Let=saw\end{bmatrix}$
当然，这个是可以嵌套的，例如“We helped her.” 
![we helped her复杂功能集图](https://img-blog.csdn.net/20170513145152210?watermark/2/text/aHR0cDovL2Jsb2cuY3Nkbi5uZXQvcXFfMzUwODIwMzA=/font/5a6L5L2T/fontsize/400/fill/I0JBQkFCMA==/dissolve/70/gravity/SouthEast)
可以看到，一个句子也是可以这样循环嵌套的。大概意思我觉得应该很明显了。
这样复杂特征集的特点就可以被归纳如下：
- 允许利用多个特征描述同一个语言单位；
- 从结构上看，复杂特征集是一种嵌套结构，可以有效地表示复杂词组或句子结构；
- 特征名的定义及其相互关系具有明显的层次性，而所有自然语言的结构都是层次性的，复杂特征集的这一特点显然对语言的层次分析有益；
- 复杂特征集便于运算，两个复杂特征集通过合一运算可以产生另一个复杂特征集，这与句法分析中词组和句子的产生是一致的。
# 4. 合一运算
合一运算主要为了运算复杂特征集的。而两个复杂特征集能够运算的前提是两个复杂特征集是相容的。下面我们给出复杂特征集相容的定义： 
若$\alpha$、$\beta$均为复杂特征集，则$\alpha$、$\beta$是相容的，当且仅当： 
(1) 如果 $\alpha$(f ) = a、$\beta$(f ) = b，且a、b都是原子，那么$\alpha$、$\beta$是相容的，当且仅当a＝b； 
(2) 如果$\alpha$(f ) 、$\beta$(f ) 均为复杂特征集，$\alpha$、$\beta$是相容的，当且仅当$\alpha$(f ) 、$\beta$(f ) 相容。
复杂集相容的定义是一个递归的定义，只有子结点都相容了，父节点才相容，这和两棵树是否全等有类似效果。合一运算的递归定义：
- 在 a、b都是原子的情况下，如果 a＝b，则 
a $\cup$b = a, 否则 a $\cup$b = $\varnothing$；
- 如果$\alpha$、$\beta$均为复杂特征集， 则 
1)  若 $\alpha$(f ) = v，但 $\beta$(f ) 的值未经定义， 
则 f = v 属于  $\alpha\cup\beta$; 
2) 若 $\alpha$(f ) = v，但 $\beta$(f ) 的值未经定义， 
则 f = v 属于  $\alpha\cup\beta$; 
3) 若 $\alpha$(f ) = $v_1$，但$\beta$ (f ) = $v_2$，且$v_1$与 $v_2$ 相容(不相抵触)，则 f = (v1 $\cup$v2) 属于 $\alpha\cup\beta$，否则, 
$\alpha\cup\beta$= $\varnothing$。
合一运算主要有2个作用：
- 合并原有的特征信息，构造新的特征结构；
- 检查特征的相容性和规则执行的前提条件是否满足，如果参与合一的特征相冲突，就立即宣布合一失败。
下面我们给出一个例子： 
复杂特征集$\alpha$: 
$\begin{bmatrix}Cat=N         \\Let=小王\\Person=Third\end{bmatrix}$
复杂特征集$\beta$
$\begin{bmatrix}Cat=N        \\Number=Sing\\Senmantic=Agent\end{bmatrix}$
合并后就为： 
$\begin{bmatrix}Cat=N         \\Let=小王\\Number=Sing\\Senmantic=Agent\\Person=Third\end{bmatrix}$
这应该很好理解，而如果这两个有一个属性相冲突，那就不能合一。我觉得这种运算是非常人性和利于理解的。如果把自然语言看作是一个传递和负载信息的系统, 并且承认自然语言中的句法成分都可以由较小的单位合并成较大的成分，那么，合一作为句法和语义分析的基本运算便是非常合理的了，因为：
(1)一个语言单位（如句子或词组等）所负载的信息可以分布在各个成分中，每个成分所负载的可以只是部分的信息；
(2)通过合一运算，在小的成分组成大的成分的过程中，小成分所负载的信息也可以同时被传递或累加为大的成分所负载的信息，在合一运算过程中，信息只会增加而不会减少；
(3)句法和语义分析都以合一作为基本运算，不仅句子的合法性可以通过语义的手段来判断，而且，还可以把句子的句法结构和语义表示用合一运算这种方式更加自然地衔接起来；
(4)对不同的复杂特征集进行合一运算，其结果同运算所进行的先后次序无关，合一的结果都是相同的。合一运算的这种无序性非常便于并行处理。
# 5. 词汇功能语法
词汇功能语法（Lexical Functional Grammar, LFG）于上个世纪70年代末期由 R. Kaplan and J. Bresnan 在美国 MIT 提出。它主要由2部分组成，这就离现代语法更近了一步。这分别是：1、成分结构层次，也就是描述句子成分的结构关系。2、功能结构层次，也就是描述句子主语、谓语、宾语等之间的关系。
它主要使用以下等式来解释语言现象：谓词的管辖范围 ＋ 谓词对论元的预示 =确定语法结构和语义解释。这样它就减轻了语法规则的作用。
准确来讲，它一方面突出词汇作用，也就是使用词为主要对象；另一方面它把功能结构的表述作为语言描述中的一个基本的独立层次，例如主语、宾语、补语等。这些都和传统的语法一致。因此，它就具有两层结构，一个是成分结构（C-结构），一个是功能结构（F-结构）。
## 5.1 成分结构
成分结构（C-结构）使用上下无关文法表示；树上的结点带有句子中词或短语所预示的功能信息，这些信息由语法规则右部的符号所带的功能注释表示，例如： 
S $\rightarrow$          NP                          VP 
         ($\uparrow SUBJ$)=$\downarrow$$\uparrow $=$\downarrow$
这个的含义就是句子 S 由 NP 和 VP 组成，其中，NP 所带的全部功能就是 S 的主语功能信息；VP 所带的全部功能信息就是 S 的谓词功能信息。简单来讲，$\uparrow $表示的是该节点在上一层所表示的功能描述，而$\downarrow$表示的是当前节点。例如：($\uparrow SUBJ$)=$\downarrow$这意思就是上一层的主语是当前节点。

这是LFG的第一套规则，即句法规则。另一套规则是词法规则，它主要标注的是词的语法功能信息，例如词的单复数等。
## 5.2 功能结构
说完了C-结构，接下来，我们来看一下功能结构（f-结构），它用以表示句子的功能关系。LFG 认为，功能结构是语言学描述上的一个独立层次，功能结构中包含语法信息，也包含语义信息，但它又不同于成分结构或语义解释。f-结构是成分结构和语义结构的中介面，既表示语法范畴之间的功能控制关系，又是建立语义结构的基础。 
上面例句的f-结构为： 
![f-结构](https://img-blog.csdn.net/20170513164147091?watermark/2/text/aHR0cDovL2Jsb2cuY3Nkbi5uZXQvcXFfMzUwODIwMzA=/font/5a6L5L2T/fontsize/400/fill/I0JBQkFCMA==/dissolve/70/gravity/SouthEast)
准确来讲，f-结构的形式有以下几个约束条件，其实看起来和复杂特征集相类似。
- f-结构是有序对的集合，每个有序对含有一个属性和该属性的值；
- 语法功能的名字或特征名字是属性；
- f-结构中的属性值有以下4类： 
1)  简单符号, 如: girl, Past, Sing 等 
2) 语义形式, 如: hand ($\uparrow $SUBJ) ($\uparrow $OBJ2) ($\uparrow $OBJ) 
3) 子 f-结构 
4) 上述三类属性的集合
## 5.3 由c-结构构造f-结构
LFG利用c-结构提供的短语结构信息和功能信息，按照严格的程序和明确的合格条件构造 f-结构。
Step-1: 从c-结构求出功能描述式(functional descriptions, 简称f-描述)； 
Step-2: 从f-描述构造f-结构。
作为连接c-结构和f-结构的中间件f-描述，它是一系列等式的集合，每个等式的形式与c-结构中各结点所带的功能注释类似，区别仅在于把直接支配元变量$\uparrow $和 $\downarrow $换成相应的结点代号(实际变量)－代真(instantiation)。实际上是将c-结构中短语结构信息和功能信息合而为一。
代真前的注释只表示树中各结点孤立的功能信息，代真后的f-描述则表示结点之间的功能关系。
代真过程如下所示： 
Step-1. 把词汇项(叶结点)的功能注释移交给父结点； 
Step-2. 用实际变量 $f_n$ 标注树的非终结结点； 
Step-3. 把 $\uparrow $和 $\downarrow $换成实际变量 $f_n$ 。
例如，前面的c-结构图经过代真后成为：

代真过程也需要合法性检验，代真过程的合法性由 LFG 中最重要的惟一性条件(uniqueness)保证，这个条件同时也是从 f-描述构造f-结构的主要合格条件。其惟一性条件为：在一个给定的f-结构中，一个属性最多只能有一个值。
第一步我们已经完成了，有c-结构转换为f-描述，使用的是代真过程。那么接下来就由f-描述转换为f-结构了。 
算法输入：f-描述中的所有等式，等式顺序没有限制； 
算法输出：f-结构 
操作:  
(1) 定位(locate) 
(2) 合并(merge) 
(3) 一致性检验： 
• 如不符合f-结构定义，定位失败； 
• 如合并时不兼容，合并操作失败。
由惟一性条件得到的f-结构的句子还不一定合法，LFG 还要进一步用一系列功能合格条件(functional well-formedness conditions)检验这个f-结构的合法性，即给定的c-结构中，句子成分的功能必须独一无二。
当然，还需要另外两个条件才能够完全的满足转换规则。 
完备性条件(completeness)：一个f-结构是局部完备的，当且仅当它包含了它的谓词所应支配的全部语法功能；一个f-结构是完备的，当且仅当它的所有子f-结构都是局部完备的。
关联性条件(coherence)：一个f-结构是局部关联的，当且仅当它所包含的全部被支配的语法功能都能被一个局部谓词支配；一个f-结构是关联的，当且仅当它的所有子f-结构都局部关联。
下面我们总结词汇功能语法的特点：
- 采用复杂特征集表达功能结构
- 合一运算作为句法-语义分析过程的基本方式
- 语法信息主要来源于词典中的词汇信息标注
- 功能结构是无序的
另外还有树连接语法和广义的短语结构语法，这里不详细介绍。
