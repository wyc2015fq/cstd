# 【基本概念】

1.数据（Data）：描述客观事物的符号，可被输入到计算机中进行处理，分为数值数据与非数值数据

2.数据元素（Data element）：又称记录，是数据的基本单位，在计算机中作为一个整体进行考虑与处理

3.数据项（Data item）：又称属性，是构成数据元素的不可分割的最小单位

4.数据对象（Data object）：相同性质的数据元素的集合，是数据的子集

5.数据结构（Data structure）：相互之间存在一或多种特定关系的数据元素的集合，分为逻辑结构与物理结构

# 【逻辑结构与物理结构】

## 1.逻辑结构

逻辑结构（Logical Structure）是指数据对象中数据元素之间的相互关系，分为以下四种：
- 集合结构：数据元素之间是**同属一个集合**的关系
- 线性结构：数据元素之间是**一对一**的关系
- 树结构：数据元素之间是**一对多**的关系
- 图结构：数据元素之间是**多对多**的关系

用示例图表示数据的逻辑结构时，将每个数据元素看做一个结点（Node），元素之间的逻辑关系用结点之间的连线表示，若关系有方向，用带箭头的连线表示

## 2.物理结构

1）物理结构（Storage Structure）是数据的逻辑结构在计算机中的存储形式，其分为以下两种：
- 顺序存储结构：将数据元素存放在地址连续的存储单元中，数据元素间的逻辑关系由元素的存储位置来表示
- 链式存储结构：将数据元素存放在任意的存储单元中，数据元素间的逻辑关系由指针来表示

2）存储空间：基本单位是字节（Byte），每一单位称为一存储单元，当按照地址随机访问任意单元时，访问不同单元时间基本相同

3）两种物理结构的特点：
- 顺序存储结构：按地址相邻，无引入附加内存
- 链式存储结构：每一数据元素分为数据域与指针域，指针域引入了附加内存，存储密度低

# 【抽象数据类型】

1.数据类型（Date Type）：性质相同的值的集合及定义在此集合上一些操作的总称，其分为两类：
- 原子类型：不可再分的基本类型，如：整型、字符型、实型等
- 结构类型：若干个类型组合而成，可以再分解，如：结构体、类、整型数组等

2.抽象：抽取出事物具有的普遍性的本质

3.抽象数据类型（Abstract Date Type，ADT）：数据模型及定义在该模型之上的一组操作，是从数据抽象得来的

![](https://img-blog.csdnimg.cn/2019030315321673.png?x-oss-process=image/watermark,type_ZmFuZ3poZW5naGVpdGk,shadow_10,text_aHR0cHM6Ly9ibG9nLmNzZG4ubmV0L3UwMTE4MTU0MDQ=,size_16,color_FFFFFF,t_70)

一个 ADT 的定义不涉及它的实现细节，在形式上可繁可简。通常包含以下内容：
- 抽象数据类型名
- 数据元素之间逻辑关系的定义
- 每种基本操作的接口（操作的名称和该操作的前置条件、输入、功能、输出、后置条件的定义） 

例如：

```
ADT 抽象数据类型名 
    Data 
        数据元素之间逻辑关系的定义 
    Operation 
        操作 1 
            前置条件：执行此操作前数据所必须的状态 
            输入：执行此操作所需要的输入 
            功能：该操作将完成的功能 
            输出：执行该操作后产生的输出 
            后置条件：执行该操作后数据的状态 
        操作 2 
            …… 
            …… 
        操作 n 
endADT
```

# 【存储密度】

存储密度(Storage Density)，用于衡量某种存储结构所占用的存储空间的大小，存储密度的值越高，说明存储空间的利用率高。

其大小是结点数据本身所占的存储量和整个结点结构所占的存储量之比，即：**存储密度 = 结点数据域所占的存储量 / 整个结点所占的存储总量**

例如：对于线性表来说，顺序表的每个结点只存放数据元素，其存储密度是 1；而对于链表来说，链表中的每个结点除存放数据元素外，还要存储数据元素之间逻辑关系的指针，其存储密度较低。
