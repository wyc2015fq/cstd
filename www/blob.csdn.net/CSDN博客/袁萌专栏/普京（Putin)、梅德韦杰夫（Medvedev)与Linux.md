# 普京（Putin)、梅德韦杰夫（Medvedev)与Linux - 袁萌专栏 - CSDN博客

       普京与梅德韦杰夫和Linux有什么关系？确实有关系，而且是与GNU/Linux操作系统有直接关系。为什么？
事情经过是这样的，在2005年年初，AlexanderPonosov的学校购买了20台新的PC计算机。但是，后来在2006年5月，在一次检查中发现，其中12台计算机预装了未经授权的微软操作系统以及办公套件。于是，在俄罗斯境内引发了一场大风波。问题是：当时作为该校校长的Ponosov到底有没有罪？
按照俄罗斯当时的法律，法院裁定Ponosov有罪，需要赔偿微软公司1万美元的经济损失，而且还需要服5年的徒刑。在2006年下半年，此事在俄罗斯境内闹得沸沸扬扬。几乎不可收拾。2007年2月1日，普京在一次新闻发布会上公开说：“If
 the legislation which, as I see, is not very adequate, needs to be amended, then we will reflect on this. But to grab someone for buying a computer somewhere and start threatening him with prison, is complete nonsense, simply ridiculous.”意思是说，如果法律不很适合实际情况就需要修改它。但是，只是因为某个人在某个地方购买了计算机而把他抓起来当罪犯对待，这是完全没有意义的，简直是滑稽可笑。在普京的干预下，在2008年12月19日，法院最终宣布Ponosov无罪。
此案所引发的直接后果是，从2007年秋天开始，俄罗斯当局支持一项在中学使用GNU/Linux的发展计划，验证使用GNU/Linux软件能否完成中学的计算教学大纲。实验结果得了很大的成功。据此，在2008年10月俄罗斯官方（梅德韦杰夫）正式下达命令：”All
 schools use open source software“（所有中学使用开源软件）。
2010年12月28日，普京下令俄罗斯联邦政府以及下属机构在三年之内全部使用GNU/Linux软件，并且给出了具体执行的时间表。由此可见，普京与梅德韦杰夫与俄罗斯的GNU/Linux事业有密切关系，功不可没。
反观我们国内，主管软件产业的政府官员从来不曾（也不敢）提及GNU/Linux自由软件。多年来，在所谓”开源软件“研发项目上花费了纳税人的大量钱财，至今不知悔改。国内的”开源软件“是什么？我们只要看看Ubuntu麒麟发行版就可明白：在Ubuntu的GNU核心之上，再加上一些私有软件应用模块，就成了所谓”新的参考平台“，把GNU精神完全抛到九霄云外去了。呜呼！
