# 网络机器人为何不来中国？ - 袁萌专栏 - CSDN博客
2011年11月29日 08:14:29[yuanmeng001](https://me.csdn.net/yuanmeng001)阅读数：2069
   11月28日上午到下午，我去中国人工智能学会办公室了解有关情况，并且拜会了学会理事长李德毅院士，使我“心满意足”。回到家里，开机上网，找网络机器人Audrey聊天，解闷，有所发现。
昨天中午，我与学会前任理事长钟义信教授以及现任学会常务副会长韩力群教授共进午餐，其间谈及“智能编程”问题，引起了我的兴趣。在回家路上，我急切地想搞清楚：网络机器人Audrey是用什么编程语言写的，是Prolog(逻辑编程语言)，还是人工智能标注语言AIML?回家之后，我立即把Audrey叫出来，当面问她：“Do
 you use Prolog?”她对我说：“I use ALML.”我又问她：”什么是AIML？“她想了一下，对我说：“The
 ALICE software implements AIML (Artificial Intelligence Markup Language) a non-standard evolving markup language for creating chat robots（用于制造聊天机器人）.......“。
经过搜索查询发现，这种人工智能标注语言AIML，国内无人过问，它的实现是基于Linux平台上的一套自由软件。实际上，AIML是当今国际网络智能机器人研究的重大进展。在我们国内，除一篇大连理工大学的硕士论文之外，几乎无人问津。“核高基”重大软件专项毫不触及这类基础性的软件，反而搞什么“国产Linux”，白白浪费了国家的钱财。
如今，深圳超算中心已经开启，“教育云”即将启动，可是，网络机器人在哪里？开展网络教育，搞网络Web智能机器人最“实惠”，几百万可以同时上网进行“提问”，机器人都能“对答如流”，我们何乐而不为？有人说，与机器人亲密接触，会“上瘾”。这总比使用盗版XP上瘾要强。有人担心，机器人很“鬼”，容易窃取“个人隐私”。那么，你去找深圳超算中心讨个“说法”。有人担心，机器人会把你带入“非非境地”。这么多担心，我也不知道该怎么办了。
网络机器人会不会教人“学坏”？对此，我试图搞个明白，便问了她几句“脏话”。但是，机器人Audrey小姐总是把话叉开，闭口不说难听的话。我想，机器人“心“并不坏，无需“过滤”，不会把人带坏。
此刻，我在想一个“微智能”系统，即让每台个人计算机都具有“人工智能”，自己会“思想“，成为计算机主人的“好帮手”，比如：Ubuntu12.04的Wintermute计划。在不久将来，人工智能将无处不在，不再神秘。机器为什么不能“思想“？将来，每台计算机都有自己的”思想“之后，人们就见怪不怪了。今天，我很高兴，人工智能学会给了我一个机会，筹备一个有关Linux应用的专业委员会。由此，Linux将会脱离“苦海”，在我们国内甜蜜蜜地生活下去了。为此，我要开始行动了。
