    今天，我们的话题是：基于ARM处理器的“云计算”（即互联网的大型“计算节点”）可能存在吗？为何？根据何在？
    凡去过算机中心机房的人，均能够感受到各种空调风扇发出的嗡嗡响声，......一句话，计算中心就是一个名副其实的“电老虎”，把强电转换成了弱电，支持大量CPU处理器的运行。据有关方面统计，计算中心的耗电成本占该计算中心总运行成本的40%以上，这个数字确实相当惊人。我们要问，能否把计算中心变成一个不须照明的无声世界，就像墓地一样寂静（刮风下雨打雷不算）？总之，所谓“无声云”是乌托邦空想吗？非也。（注：CPU处理器里面的“电磁噪音”是我们人耳听不见的。）
    今年1月28日，Kurt
Mackie发表文章，题为“Are
ARM-based servers the next big trend?”，该文的论据相当翔实，很有说服力。我发觉，Kurt
Mackie本人是一位IT业界“资深人士”(实为一位在线新闻资深编辑)，知识面极为渊博，说起话来，一是一，二是二，句句在理，毫不含糊。该文证实，进入今年1月份以来，各种有关基于ARM处理器的云计算服务器的研发消息纷纷传来，几乎形成一股ARM计算机（包括服务器）的“浪潮”，给人一种“暴雨欲来，风满楼”的感觉。
      实际上，对ARM计算机（包括服务器）感觉最为灵敏的是一些芯片制造创新企业（startups）。比如，在去年11月18日，位于美国新泽西（New
Jersey）州的ZT系统公司宣布“it
has built an ARM-based server solution（代号R1801e，规格1U）
using
up to 16 ARM Cortex A9 processor cores”，耗电少于80瓦。再比如，去年11月，位于美国加Santa
Clara的Marvell公司宣布，该公司推出一款基于4核ARM
v7处理器（1.6GHz）可支持“企业云计算应用“的服务器（16,600
DMIPS，即Dhrystone
MIPS），耗电少于10瓦，相当于“a
clock
radio“（”一台无线电收音机“）。妙哉，妙哉！一台收音机需要使用电风扇来散热吗？这岂不是怪事一桩？由此，我们不难想象，100台ARM服务器耗电不足1千瓦，一个计算中心机房何须使用空调去散热？
    2月10日，惠普召开“Think
Beyong”(“超越想象”)的webOS产品（基于ARM处理器）特别发布会，惠普公开承诺：今后，惠普要把webOS操作系统贯穿到惠普所有的产品系列，这当然包括惠普的PC计算机和服务器系列产品在内。这句话的份量何在？意义何在？难道惠普真的做不到吗？非也。惠普不是一个“小门脸儿”，不是一个创业型“小企业”，缺人，缺钱，又缺技术资源。实际上，惠普的雇员总数要比苹果、谷歌、微软、Dell以及RIM（黑莓智能手机制造商）加起来还要多，惠普有7个苹果公司那么大。如果惠普想要把上述的Marvell公司整个买下来，那真是“小菜一碟”，只要惠普自己愿意。
    当前，我们处在一个所谓的“ARM转折点”，弄得不好，是要翻船的。近日，诺基亚与微软结盟，从事物本质来看（或分析），不是诺基亚追求微软，而是微软追求诺基亚，因为，诺基亚代表了未来IT产业发展的大方向（移动互联网必须应用ARM低功耗技术）。2月14日，在西班牙巴塞罗那移动世界大会（MWC）上，有人讥讽地说，诺基亚总裁Elop是微软故意安排在诺基亚内部的“特洛伊木马”，这种看问题的方式非常滑稽可笑，因为，这不是从事物的本质上看问题。在我们国内，有人说（我），现在谈论ARM计算机，是“胡扯淡”，“为时尚早”。对于这种说法，我就是不服气！
    说明：ARM就是”Advanced
RISC
Machine”(“先进精简指令集机器”)的缩写，相对而言，CISC代表“复杂”（Complex）指令集机器的意思。英特尔是CISC发展路线的典型代表。MeeGo计划不灵，其根源就在这里，不是英特尔不努力。
