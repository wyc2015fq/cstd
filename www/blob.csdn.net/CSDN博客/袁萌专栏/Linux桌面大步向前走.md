    现今，谷歌Android表面上看起来似乎很“繁荣”，各种“数字、指标”满天飞，实际上，麻烦不断，危机四伏。此话当真？
    在十年前，2000年11月2日，《自由软件基金会》发起人Stallman曾说过这样的话：“software
source code should always be public and open to change so that it can
continually be improved by
others.”我们说，Linux桌面，尤其是GMOME桌面，走的就是这条开发路线，从未动摇过。Stallman还表示，（这条发展路线能够保证）：“so
that the desktop operating system market is not controlled by a
single
vendor.”坦白地说，谷歌Android的发展路线从来都不是这样的。谷歌Android被关在谷歌家门里面封闭搞开发，Android的掌控权完全控制在谷歌一家公司手中。开源与闭源，谷歌独家说了算数。谷歌Android的追随者，现在是该清醒的时候了。
    前天上午，朋友送我一块自启动U盘，上面装有GNOME桌面（3.0
beta，正式版定于4月6日发布）。我启动进去一看，眼睛顿时一亮，这就是Web
2.0的桌面风格啊！特别值得注意的是，GNOME
3.0与Ubuntu
11.04（4月28日发布）的外貌几乎“一模一样”，根本不用下拉菜单，手指一点图标就“得”（即O.K的意思）。大家知道，Ubuntu
11.04没有直接使用GNOME
3.0(因其体积大了一些)，而是采用了Unity桌面（也是一种Linux桌面），特别适合用于智能手机。实际上，从Ubuntu
10.04新版本开始，所有Ubuntu计算机桌面的风格都一致了，智能手机、平板电脑、笔记本和台式机，人机界面都一个“模样儿”，都是Unity界面。也就是说，任何人只要熟悉了一种Ubuntu计算机界面就行了。这是一种什么“劲头”？谷歌说什么Android
3.0版本是专门针对平板电脑设计的，还不适合用于智能手机，所以不开放源代码，这全是借口。
    据说，今年的GNOME年会现在正在印度举行，决定GNOME的下一步发展方向。经过近十年的努力，Linux桌面GNOME
3.0才得以最终发布（4月6日），而Ubuntu
11.04新版本也正好在4月份发布。可以说，今年4月份是Linux桌面发展史的一个“转折点”，这个月份是一个Linux桌面发展的“里程碑”。相对而言，谷歌Android只是一个“小儿科”（3～4岁），“海市蜃楼”而已。
谁的心中都明白一个道理，在未来，智能手机的市场空间很大，而台式机快要市场饱和了。转向智能手机市场是不可避免的，但是，不一定要吊死在谷歌Android（暴发户）一棵树上。Ubuntu
11.04就要发布了，它的源代码是不会关闭的。我们要走Linux桌面的正道，而不要走Linux桌面的邪道。谷歌Android走的是与Linux内核“分叉”（“fork”)的邪道，结果呢？到处碰壁，吃了官司，危机四伏，麻烦不断。这就是“走邪道”留给我们的历史教训。
