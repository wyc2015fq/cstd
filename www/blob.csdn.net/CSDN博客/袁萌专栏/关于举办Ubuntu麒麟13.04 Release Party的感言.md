# 关于举办Ubuntu麒麟13.04 Release Party的感言 - 袁萌专栏 - CSDN博客

    当前，关于Ubuntu麒麟的未来走向（或动向）是一个十分敏感的重要话题。对此话题，我考虑了很久，觉得有话要说，不吐不快，但是，需要特别慎重，要有事实作为依据，不可乱说乱动也。
5月11日，Ubuntu麒麟开发团队将在北京举行UbuntuKylin
 13.04发布的庆祝活动（即”Release Party“），届时，请来金山Office技术负责人做所谓”主题报告“。此举符合当今的”时宜“吗？也就是说，大方向对头吗？
4月25日，UbuntuKylin
 13.04 GNU/Linux发行版的发布给我们送来了LibreOffice 4.0。这是我们看问题的出发点。2010年9月28日，LibreOffice项目诞生。随后，在国际《文档基金会》的正确领导下，该项目得以迅速地发展起来。对该项目而言，什么是”正确的领导“？在《文档基金会》关于这个4.0版本发布的公告中对外宣称：由于该项目的管理方法（Governance）和授权规则（GNU
 Copyleft）的正确性吸引了许多开发者加盟该项目，现在已经有500多人参与项目开发，其中3/4是志愿者，而且，该项目每半年有一次重大升级。
大家知道，2013年1月29日，微软发布了Office
 2013，全面支持开放文档格式（ODF），包括文件的读(read)、写(edit)与存储(save)。这说明了什么呢？这充分说明了开放文档标准（ODF）的巨大威力，微软最终向左（ODF）转，而不是向右转。实际上，开放文档格式就是LibreOffice开发团队所始终坚持遵循的文件格式标准。也就是说，实际上是微软靠向（兼容）LibreOffice，而不是相反。在这种情况下，金山大谈”100%兼容微软Office”还有什么实际意义呢？
从实质上来看，LibreOffice4.0的真正“转向”是其图形接口（API）转向更加适合于移动互联设备的开发者（或开发商），比如，平板电脑与智能手机领域的开发者，这与Ubuntu的Unity计划是不谋而合，完全合拍。我们注意到一种发展趋向，”LibreOffice
 is moving to a new graphics stack.This is based on GtkBuilder, which builds interfaces from an XML user interface (UI)definition”。这种发展的大趋势是很容易理解的。
坦率地讲，LibreOffice开发项目是当今全球最大的独立自由软件（GNUCopyleft）开发项目，其代码量最大。它的实际走向值得我们关注。5月11日举行的“Release
 Party”庆祝活动应当围绕这个重点进行，不要被金山的“主题报告”带到错误的方向。
说明：我认为，在XP盗版成灾（而且根深蒂固）的国情下，桌面软件没有未来。在移动互联的前景下，我们的软件业还有一“丝”（比线还细）希望。
