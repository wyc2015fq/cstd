# Linux 2.6.38发布有感 - 袁萌专栏 - CSDN博客
2011年03月20日 06:11:00[yuanmeng001](https://me.csdn.net/yuanmeng001)阅读数：1863
         3月15日，Linux 2.6.38发布了，很平静，国内似乎没有什么欢呼声。我的心情也很郁闷。何故？
    
        我们知道，Linux已经搞了20年了，进程调度的优化问题至今才得以很好的解决。有人给Linux的进程调度（或算法、实现方式）打了一个小补丁，被人们称为“wonder patch”（“神奇补丁”），竟然使得Linux桌面环境在某些情况下对外界的反应速度加快了很多（considerably faster），真是匪夷所思也。这个新版本重新构建了“虚拟文件系统”（VFS），处于应用程序与文件系统代码之间（好像是个“中间人”），在文件读写操作过程发挥了更大的作用，文件操作性能得以提高许多，比如，在根目录里面，进行文件系统查询（find命令）速度可以提高30%。该版本能够支持更多的新型图形处理芯片组（AMD与英伟达），......提高了Wi-Fi的性能，......还有其他许多性能提升，在此我就不说了。
    
        近日，谷歌Android违反GPL规则一事，闹得业界沸沸扬扬。对此，有人心里面很不痛快，总想“反扑”过来......谷歌很聪明，用自己发明的一套“程序”手段替Linux内核模块一一做”代码检查”，竟然对一些Linux”模块”标注上“无版权信息”，实在可气也。我们退一步讲，即使某些Linux模块没有版权信息，也不能把该模块放在你谷歌的名下啊！实际上，谷歌这是在玩“邪”的。谷歌Android的出发点虽然不错，想把自由软件（国内有人叫“开源软件”）应用到智能手机领域中去，但是，谷歌塞进了一些“私货”，也不向社会公众说个明白，把事情搞砸了。3月14日，美国《网络世界》记者Jon Brodkin询问自由软件老祖宗Richard Stallman一个问题：自由软件有什么成就？Stallman的回答是，the first thing was Android -- not Google's version of Android, but rather a third-party version of the mobile OS in which all proprietary software has been stripped out（被剥离）。他的意思是说，移动操作系统Android算是一项自由软件的成就，但是，不是谷歌的那个Android版本，而是剥离所有私有软件的那个移动操作系统。由此可见，Stallman对谷歌Android颇有微词，同时，他也指出了移动操作系统的发展方向。
    
       Stallman说，有一种Android版本，叫做“Replicant”，其中没有任何私有软件，能够运行在HTC的“Dream phone”智能手机上。他听说，这东西在欧洲国家已经能用了。结合我国情况，与Replicant手机相比，联通的沃Phone还差一大节。
    
       我认为，商业软件（多半是私有软件）也是个好东西，因为，广大客户也喜欢它。但是，私有就是私有，自由就是自由，这一点你要对社会公众交代清楚，不能让私有软件占自由软件的“便宜”，像谷歌那样装“糊涂”。打个比方，惠普的WebOS就是基于Linux 2.6.34内核（对于这一点，惠普毫不含糊其词，占Linux的“小便宜”）构建的一款惠普名下的“Linux产品”，另外增加了一些原Palm的“私有模块”，否则，Palm怎能卖出12亿美元的好价钱？惠普的全球“云战略”能够把“原生态”Linux内核推向全球，这有什么不好呢？......Stallman认为蜂窝手机会暴露用户的行踪(被跟踪)，至今偏执地坚持不用手机，此举也太固执（离谱）了。我上街，家人一定叫我带上手机，以便随时好找到我。......Stallman至今光棍一个，没有人管着他，随他去了。
