     近日，《NoSoftwarePatents》奠基人Florian
Mueller揭露谷歌Android系统的关键部件（the
Bionic Library）非法拷贝Linux内核头文件内容，致使整个Android生态系统面临全线崩溃的危险境地。具体情况怎样？
     3月18日，Jon
Brodkin在“Android
violates Linux license”一文中，引用Mueller的一段文字如下：
"Google
copied 2.5 megabytes of code from more than 700 Linux kernel header
files（头文件）
with a
homemade program that drops source code comments and some other
elements, and daringly claims (in a notice at the start of each
generated file) that the extracted material constitutes 'no
copyrightable
information'"，意思是说：......谷歌大胆地宣称：这里所引用的（Android）2.5MB材料”没有可版权的信息”。实际上，相关的版权信息早被谷歌自己删去了。谷歌做贼的尾巴终于被抓住了！
    Mueller指出：如果发生法律诉讼而且谷歌输了官司，那么，“the
development ecosystem built around the hugely popular Android mobile
operating system would be at risk”，意思是说，基于Android的生态系统将要面临风险（“at
risk”）。Mueller最后指出：如果谷歌不能改正这些错误，那么，这个风险就意味着："collapse
of the Android ecosystem“（”Android生态系统的崩溃“）。
    大家知道，谷歌Android的程序代码的来路有点儿“野”（不正的意思），那么，今后就别拿谷歌的Android到处去说“事儿”。甲骨文控告谷歌偷窃Android系统Dalvik虚拟机源代码，这是铁板钉丁的事情，谷歌是赖不掉的。中国联通自主搞沃Phone系统完全避开Android是非常明智的。我感到很奇怪的事情是，对于Android而言，谷歌总是躲躲闪闪，不说Android是基于Linux内核的衍生品。这里面有其商业目的。我们的“国产”Linux，恐怕与此类似。
