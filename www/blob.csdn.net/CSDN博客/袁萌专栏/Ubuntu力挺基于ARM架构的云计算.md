# Ubuntu力挺基于ARM架构的云计算 - 袁萌专栏 - CSDN博客
2011年10月09日 04:33:54[yuanmeng001](https://me.csdn.net/yuanmeng001)阅读数：972
    2010年4月28日，ARM公司首席执行官Warren
 East对《EE Times》说：“servers
 based on ARM multicore processors should arrive within the next twelve months”，意思是说，在未来12个月之内，基于ARM多核处理器的服务器将会出现。事实已经证明：近一年来，ARM服务器的开发出现了惊人的巨大进展，低功耗、高性能的“云计算”已经呼之欲出。这是一个重要概念或认识：基于ARM架构的低功耗服务器已经浮出了水面。
我们要问：基于ARM多核处理器的“物理云”（即互联网“云中心”）怎么管理？由分布在不同“云中心”的数十万台ARM架构服务器所组成的“云”，如何管理运行起来？定于10月13日发布Ubuntu
 11.10(服务器版本)是世界上第一款支持ARM多核服务器的Linux操作系统，请见短文：“Ubuntu
 Server Trenches, The Big Picture“（10月6日发表），其具体说法是“Ubuntu
 Server 11.10 will be the first release with support for the ARM architecture“，该文还说：“We’re
 now at a point where ARM processors are able to handle workloads traditionally done by Intel and AMD based servers“，意思是说，我们现在处于一个”时间点“上：即ARM多核处理器（架构）能够应付那些原来需要传统英特尔与AMD处理器所完成的工作负载。这又是一个很重要的基本认识。
昨日下午，我与深圳的哪位朋友（“云老板”）一同到位于北京上地的IBM研发中心观看IBM“内部云“的实际演示，提高了对云的思想认识，纠正了某些错误观念，比如，传统云是一只“电老虎”，而ARM云则代表了未来的发展方向。在此期间，我把那只标有乌邦托品牌的”云终端“（”Cloud
 Terminal“）拿给IBM技术专家看（简单测试）了一下，原来那里面有ARM处理器（533MHz），内存64MB，自带一个很小的操作系统，可以自启动，直接访问互联网云中心，由此，我们理所当然地可以称其为“云终端”了。
记得，我在某网页上曾看到一台ARM多核服务器耗电只有60瓦，是普通x86服务器的十分之一，由此，互联网云中心“电老虎”的帽子就要被摘掉了，云中心机房的空调散热问题基本上也算是解决了。实际上，定于明年4月26日发布的UbuntuServer
 12.04 LTS版本将是一款更为丰满（a
 full blown）的基于ARM架构的服务器云操作系统，对我们而言，这也算是一种”福音“吧！
说明：昨日，在IBM研发中心观看“云演示”归来之后，我更加坚定了一种信念：在不久的将来，“云终端”的实际应用前景几乎是“无限制”的，这就是绿色节能IT产业的美好未来。
