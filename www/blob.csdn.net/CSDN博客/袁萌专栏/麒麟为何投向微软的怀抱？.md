# 麒麟为何投向微软的怀抱？ - 袁萌专栏 - CSDN博客
2011年08月25日 07:31:23[yuanmeng001](https://me.csdn.net/yuanmeng001)阅读数：1569
    在古代，麒麟（kylin）代表祥瑞的吉祥物。在当今，麒麟操作系统是中标软在2010年下半年才发布的作为我国863计划重大软件专项的研发成果。但是，在8月23日，不知怎么搞的，麒麟却投向了微软的温暖怀抱。中标软的头脑是否发晕了？
8月23日，MaryFoley发表署名文章，题为“Microsoftsigns
 'legal covenant' with Chinese Linuxvendor“，此文刚一发布，立即被全球媒体引用、转载3,800多次，几乎无人不知。文中有一段话，很让人生气：“Iasked
 Microsoft whether the “legal covenant agreement”（法律协议）isthe
 same kind of patent-protection deal that Microsoft has forgedwith other Linux vendors, including
[Novell/AttachmateSUSE](http://www.zdnet.com/blog/microsoft/microsoft-and-suse-extend-microsofts-controversial-novell-linux-pact/10164),[Amazon](http://www.zdnet.com/blog/microsoft/amazon-becomes-the-latest-company-using-linux-to-pay-microsoft-for-patent-deal/5346),[GeneralDynamics,
 Onkyo and Velocity Micro](http://www.zdnet.com/blog/microsoft/microsoft-gets-android-tablet-maker-onkyo-to-sign-sign-a-patent-agreement/9888) — via which these vendorsare paying Microsoft patent royalties on their Linuximplementations”，虽然微软的这位发言人不愿意这样说（“declined
 to say”），这里暗示，根据这份法律协议，“thesevendors
 are paying Microsoft patent royalties on their Linuximplementations”，即向微软缴纳专利使用费。
我深感不解的是，我国的863计划软件重大专项在微软Windows服务器平台上跑一跑，为什么还要向微软缴纳专利费？微软是不是太欺负人了？反过来又一想，我们的麒麟为何投向微软的怀抱？在微软的Hyper-V怀抱里，有许多东西，比如：WindowsServer
 2008 R2 with Hyper-V; System Center Operations Manager; SystemCenter Virtual Machine Manager; System Center Service Manager; Opalis(workflow automation); and System Center Virtual Machine Manager R2Self-Service Portal，等等，等等，好温暖啊！
不久之前，永中Office的核心专利技术落到微软派系唐敏与刘明的手中，如今，麒麟又投向微软的怀抱。我们的国家是怎么了？
