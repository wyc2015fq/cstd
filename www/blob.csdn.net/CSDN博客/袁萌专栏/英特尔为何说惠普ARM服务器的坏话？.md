# 英特尔为何说惠普ARM服务器的坏话？ - 袁萌专栏 - CSDN博客

    由于众所周知的原因，英特尔对惠普近日推出的ARM服务器大为“光火”（发怒、生气），认为惠普抢了自己的“风头”。英特尔是怎么说的呢？
11月4日，Jon
 Stokes在《Wired Tech》（“连线技术”）发表文章，题为“Intel
 Responds to Calxeda/HP ARM Server News: Xeon Still Wins for Big Data”，披露了一些实际情况。该文说，英特尔服务器部门公共关系（PR）负责人Radek
 Walcyzk对惠普推出ARM服务器一事作出了英特尔的官方回应，借此，英特尔希望社会大众知道一个事实：自2009年，英特尔也有类似的低功耗微型服务器（“Microserver”）的开发计划（基于Xeon处理器），比惠普ARM服务器还要早一些。意思是说，英特尔也看中了“微型服务器”节能的优势，同时，英特尔表示，也有一些其他的服务器厂商推出了基于x86处理器的“微型服务器”产品。简而言之，在微型服务器市场化方面，英特尔比惠普还要起步早一些。英特尔就是要争这个“第一”，维护所谓“x86软件生态系统”的既得利益，这就是英特尔所持的“立场”。（注：惠普是英特尔X86芯片的最大客户，如今造反，英特尔当然不容。）
根据英特尔自称的两年来接触“微型服务器”的实际经验，英特尔认为，惠普ARM服务器性能不如基于x86处理器的“微型服务器”（更经济），特别是在处理大数据负载的情况下，......转弯抹角地说惠普ARM服务器的坏话。
我们应该怎么看待惠普ARM服务器呢？实际上，惠普近日推出的这款ARM服务器不是简单的ARM“片上系统”的“堆积物”，而是采用了Calxeda公司的最先进“链接技术”将大批”ARM片上系统“集成为一个“整体”，统一供电、统一冷却，统一管理，使其工作效率大大提升，达到实用水平。
11月2日，Calxeda总裁Barry
 Evans对媒体表示：“each
 individual ARM processor is not necessarily a fast chip, coming in at a clock speed of up to 1.4 Ghz."Is it blazingly fast? No," Evans said. "Is
 it fast enough?Yes."他的意思是说，单个ARM处理器不必是最快的芯片，但是，当芯片时钟频率升至1.4GHz时，”它是闪光般的快吗？非也。“他又接着说：”这时，它是足够快吗？Yes.“总之，ARM服务器不是万能，它适用于“offline
 analytics, web applications (including storage and file serving apps），这已经足够了。
我们要注意的是，在云计算时代，惠普ARM服务器为Linux扩大应用范围提供了一个非常难得的机会。站在Ubuntu背后的人们就紧紧地抓住这一机会不放手，与Win8抢时间，争市场。实际上，涉及ARM服务器的软件开发空间相当大，我们绝不可随意丢弃，跟在英特尔屁股后面“小跑步”。实际上，虽然x86服务器立下了历史性的”汗马功劳“，但是，现在该是ARM服务器登场的时候了。在信息产业发生重要转折的关头，我们不能闭着眼睛睡大觉（或是打桥牌无聊）。是英特尔没有技术水平吗？非也。英特尔是被X86生态系统的既得利益拖住了后退，动弹不得，以致看见了ARM就生气。
