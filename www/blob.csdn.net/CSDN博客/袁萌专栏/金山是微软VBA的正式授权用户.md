# 金山是微软VBA的正式授权用户 - 袁萌专栏 - CSDN博客
2013年05月12日 03:53:25[yuanmeng001](https://me.csdn.net/yuanmeng001)阅读数：646
     5月11日下午2点，在位于北京中关村的中国科学院大学（原来的研究生院）教学楼S201教室，举行Ubuntu13.04发布庆祝活动（ReleaseParty），与会者超过300余人，各界人士见证了这一事件。
会上，金山Office架构师之一金持重先生做了主题报告，当众宣称：金山购买了微软的VBA授权。此话顿时引起我的注意，我怕自己耳朵听错了，便立即询问坐在我旁边的同事，得到了肯定的回答。什么是VBA授权呢？
微软的VBA就是”Visual
 Basic for Applications“的简称，就是：”an implementation of[Microsoft](http://en.wikipedia.org/wiki/Microsoft)'s [event-driven programming](http://en.wikipedia.org/wiki/Event-driven_programming) language“，意思是说，VBA是微软事件驱动编程语言的实现，它联系着一种微软私有的集成开发环境（IDE）。准确地说，“VBA
 enables building user defined functions, automating processes and accessing [Windows API](http://en.wikipedia.org/wiki/Windows_API) and other low-level functionality through [dynamic-link libraries](http://en.wikipedia.org/wiki/Dynamic-link_library)”。这就是说，借助动态链接库金山可以访问微软Windows应用程序接口（API）以及其他底层函数。
什么是VBA的正式授权协议？也就是说，金山公司必须遵守微软的EULA（终端用户授权协议）的全部条款。也就是说，在每一套金山Office软件包产品里面都有微软VBA的授权条款因素，为此金山必须支付微软相应的授权费用。如此看来，金山高举的所谓”民族软件“大旗其实只不过是一个愚弄社会公众的“幌子”而已。
由此我们可以看出，金山公司之所以敢于公开宣称：金山忠实可靠地”克隆“了微软Office办公套件，其实际根据就是一条：金山是微软VBA的正式授权用户（永远不会”开源“，也不能”开源“，受到微软EULA条款的制约）。如今，UbuntuKylin（麒麟）与Kingsoft（金山）打得火热是不可理解的。
