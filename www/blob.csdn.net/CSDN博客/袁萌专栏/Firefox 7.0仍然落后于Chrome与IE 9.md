     9月27日，浏览器Firefox7.0发布，极大地改善了内存使用方法（Vastlyimproved
 memory usage (up to 50% better)，提高了运行速度。但是，相比之下，不容乐观。
9月27日，美国《PCMagazine》当日刊登评论文章，题为“Firefox7.0”，该文肯定了其上述优点之外，同时指出：在某些基准测试方面，“Firefox7.0
 Still trails Chrome and IE9”，意思是说，在某些方面，Firefox7.0仍然落后于谷歌的Chrome浏览器与微软的IE9，据此，编辑部对其评分是4分(Verygood)，读者对其评分是3.5分（Good）。
公平地说，火狐浏览器开发团队为7.0这个新版本的发布做出了不少的努力（或贡献）。从Mozilla基金会官方网站（FirefoxBugFixes）上，我们可以看出，为了这个新版本的发布，开发团队共计改正了近1200个BUG，费时费力，很不容易。这些BUG都有ID号码，比如：661518，其内容是：“homepage
 is not repainted correctly when it comes back infocus”。修改一BUG，谈何容易？
9月17日，谷歌浏览器Chrome14发布了。前几天，我在孩子的计算机里面偶尔发现，他已经用上了Chrome14。我上去试了一试，乖乖，好快啊！我等着10月13日Ubuntu11.10新版本发布之后，除了缺省的Firefox7.0之外，再装一套谷歌的Chrome14浏览器体验一把，因为，目前我正在为云桌面eyeOS平台寻找合适的“配套”浏览器。
此刻，我很担心的是：因为我在此文标题中又说了火狐浏览器7.0版本的坏话，说它还不如微软浏览器IE9的某些性能，恐怕又会招来骂声。火狐浏览器不如微软的IE9，这是客观存在的事实，我们必须面对（或承认）。回顾以往，在2009年8月26日，网上突然出现一则消息，题为”无间道被识破？袁萌被COPU解除联盟副秘书长职务“（宋可为发），消息具体内容是“自即日起，解除袁萌副秘书长职务“，后面盖着中国开源软件推进联盟的公章，还有原中国开源软件（OSS）推进联盟主席陆首群的亲笔签字，阵势怪怕人的。在当时，对此决定，我很生气。什么是”无间道“？实际上，这里所谓的“无间道”指的就是微软的“卧底”。我怎么成了微软的“卧底”（间谍）？简直是莫名其妙！为此，在今年9月27日下午1时许，我到中国开源软件推进联盟联盟秘书处进行“申诉”，喊冤，要求给个公平的“说法”。
