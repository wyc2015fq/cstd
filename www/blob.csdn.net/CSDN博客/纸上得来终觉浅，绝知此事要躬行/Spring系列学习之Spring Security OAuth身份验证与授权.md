# 概述

Spring Security OAuth使用标准的Spring和Spring Security编程模型和配置惯例，为使用Spring Security with OAuth（1a）和OAuth2提供支持。

## 特性
-     支持OAuth提供程序和OAuth使用者
-     Oauth 1（a）（包括两条腿OAuth，a.k.a。“Signed Fetch”）
-     OAuth 2.0



将安全性应用于应用程序并不适合胆小者，OAuth也不例外。在开始之前，您需要确保了解OAuth及其旨在解决的问题。 [OAuth网站](https://oauth.net/)上有很好的文档。您还需要确保了解[Spring](https://spring.io/projects/spring-framework)和[Spring Security](https://spring.io/projects/spring-security)的工作原理。


您将希望对[OAuth](https://oauth.net/)（和/或[OAuth2](https://tools.ietf.org/html/rfc6749)）和[Spring Security](https://spring.io/projects/spring-security)非常熟悉，以最大限度地提高此开发人员指南的有效性。 Spring Security的OAuth与这两种技术紧密相关，因此您越熟悉它们，就越有可能识别所使用的术语和模式。

## 快速开始

使用[Spring Initializr](https://start.spring.io/)引导您的应用程序。
