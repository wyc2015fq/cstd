# 概述

## 生命结点

该平台将于2019年4月9日达到其支持寿命的终点。布鲁塞尔和开罗线路的维护将在此之前继续发布。鼓励平台用户开始使用Spring Boot的依赖管理目录，可以使用spring-boot-starter-parent作为Maven项目的父项，也可以导入spring-boot-dependencies bom。

## 概览

Spring IO是一个用于构建现代应用程序的有凝聚力的版本化平台。它是一种模块化的企业级分发版，可提供一组策划的依赖项，同时使开发人员完全可以控制仅部署所需的部件。 Spring IO是100％开源，精简和模块化的。

## 特性
-     一个平台，许多工作负载 - 构建Web，集成，批处理，响应或大数据应用程序
-     使用Spring Boot极大地简化了开发体验
-     开箱即用的生产就绪功能
-     精心协作的协调依赖项
-     模块化平台，允许开发人员仅部署他们需要的部件
-     支持嵌入式运行时，经典应用程序服务器和PaaS部署
-     仅取决于Java SE，并支持Groovy，Grails和一些Java EE
-     适用于您现有的依赖管理工具，如Maven和Gradle
-     Spring IO平台经过认证可用于JDK 7和8 [1]



1.虽然Spring IO平台支持JDK 7和8，但许多单独的Spring项目也支持较旧的JDK版本。有关具体的最低要求，请参阅[各个项目的文档](https://spring.io/docs)。
