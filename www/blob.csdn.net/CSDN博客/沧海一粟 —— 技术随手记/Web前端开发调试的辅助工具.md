# 介绍

在开发Web应用前端时，总是免不了要在浏览器中进行跟踪调试。以Firefox为例，有很多插件，可以让我们的工作事半功倍。

# Firebug



![](https://img-blog.csdn.net/20131113102315562?watermark/2/text/aHR0cDovL2Jsb2cuY3Nkbi5uZXQvZWVuZ2Vs/font/5a6L5L2T/fontsize/400/fill/I0JBQkFCMA==/dissolve/70/gravity/SouthEast)


最强大的前端开发调试工具。可以查看编辑HTML、CSS、JavaScript，查看HTTP请求，并直接输入执行JavaScript代码。

# Clear Cache

![](https://img-blog.csdn.net/20131113102320609?watermark/2/text/aHR0cDovL2Jsb2cuY3Nkbi5uZXQvZWVuZ2Vs/font/5a6L5L2T/fontsize/400/fill/I0JBQkFCMA==/dissolve/70/gravity/SouthEast)


一键清空Cache。我们在调试代码的时候，经常碰到这样的情况：明明代码已经改了，可无论怎么刷新页面，新的效果就是出不来。于是就采用各种手段去诊断到底哪里出了问题，比如设断点调试。但这个往往很耗费时间。有时候不记得代码改哪里了，或着改的地方太多，这种调试简直就是不可能完成的任务。终于费了九牛二虎之力，发现问题仅仅是因为某个文件没有加载到最新版本。很简单的问题，却经常让开发者阴沟里翻船。

每次刷新页面前，养成先点ClearCache的习惯，就可以避免这种问题。

# 搜索引擎

![](https://img-blog.csdn.net/20131113102327953?watermark/2/text/aHR0cDovL2Jsb2cuY3Nkbi5uZXQvZWVuZ2Vs/font/5a6L5L2T/fontsize/400/fill/I0JBQkFCMA==/dissolve/70/gravity/SouthEast)


个人的知识不是万能的，搜索引擎才是万能的！

代码调试过程中碰到解决不了的问题，求助搜索引擎。最常用的自然是Baidu和Google。这个插件省去了用户输入URL打开页面的步骤，直接输入关键字回车就可以得到相应的搜索结果，非常方便。还可以通过管理搜索引擎创建自己的搜索引擎列表。

# QuickProxy



![](https://img-blog.csdn.net/20131113102324625?watermark/2/text/aHR0cDovL2Jsb2cuY3Nkbi5uZXQvZWVuZ2Vs/font/5a6L5L2T/fontsize/400/fill/I0JBQkFCMA==/dissolve/70/gravity/SouthEast)


说到Google，不得不说没有代理简直没办法用。而Baidu不用代理访问快。因此我们也来一个一键打开/关闭代理的插件：QuickProxy。首先在浏览器设置里面填好代理信息，然后平常就用不上它了，完全用QuickProxy控制。点击按钮变绿色时打开代理，变红色时关闭代理。



# 总结

只要细心发掘，总能找到一些自己趁手的工具，帮助我们更好地工作。而上面的这些浏览器插件组成了目前我开发Web应用前端时的“工具包”。既能帮助我提高工作效率，又没有多到影响正常的调试工作。
