**为什么要用框架？**1. 团队快速开发

2. 代码规范性

3. 安全性，框架一般会做一层安全处理

4. 代码可扩展和可维护


**但是非得拘泥于框架么？**1. 不能局限于使用框架，只会使用框架必然会成为机器人。

2. 需要去了解框架的结构，本质，以及实现方式。吸取优秀的地方。

3. 不要迷信框架，框架不能帮你解决所有的问题，也不能保证没有BUG，出现问题应该学会去思考怎么解决，是修改框架还是修改实现方式。、


**如何选择框架？**1. 如果你读不懂框架里面的设计，代码细节，那么请放弃吧，真做项目了，痛苦不断的。

2. 团队至少有一半以上的人对框架很熟悉，否则请放弃吧。

3. 不要轻易相信网上很多人说 哪个框架不错 没做过调研就用了，很多网上的人说的 估计他自己都没用过，人云亦云，就好比我见过好几个团队在zend Symfony等框架上痛苦挣扎，这些都是5年+的老程序员了，所以选用的时候慎重再慎重，选框架很轻松，但是痛苦是长期的！

4. 反正一句话，你能读懂框架，使用方便，能修改，团队都能接受，框架功能方面还可以，那么大概就能选用了！](https://so.csdn.net/so/search/s.do?q=zend&t=blog)](https://so.csdn.net/so/search/s.do?q=php&t=blog)](https://so.csdn.net/so/search/s.do?q=框架&t=blog)
