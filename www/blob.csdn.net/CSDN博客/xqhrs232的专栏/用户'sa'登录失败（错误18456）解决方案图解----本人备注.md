## 用户'sa'登录失败（错误18456）解决方案图解
    当我们在使用sql server 的时候可能会遇见这样错误提示“无法连接到.,用户'sa'登录失败（错误18456）”,
    如图：
![](http://blog.itpub.net/attachment/201403/13/24041331_13947108111j74.png)
具体的解决方案：
- 
首先使用管理员身份打开SQL server ，并且以windows身份进入。
- 
然后打开“安全性”--“登录名”中找到用户“sa”打开其属性面板。
如图：
![](http://blog.itpub.net/attachment/201403/13/24041331_1394710811ZecG.png)
- 
接下来选择“常规”设置密码，并且点击“确定”保存。
如图：
![](http://blog.itpub.net/attachment/201403/13/24041331_1394710811rpPF.png)
- 
然后选择“状态”--登录设置为“启用”，并且点击“确定”保存。
如图：
![](http://blog.itpub.net/attachment/201403/13/24041331_1394710811ire0.png)
- 
然后再打开sql服务器属性页面操作是“选择服务器右击--属性”，选择“安全性”将服务器身份验证选择为“SQL server 和 Windows 身份验证模式（s）”，点击“确定”保存。
如图：
![](http://blog.itpub.net/attachment/201403/13/24041331_13947108122i7x.jpg)
- 
最后再右击服务器选择“重新启动（A）”，将服务器重新启动就OK 了。
如图：
![](http://blog.itpub.net/attachment/201403/13/24041331_139471081236u0.jpg)
假如以上的办法还是不能解决就再进行一下的操作：
打开 “开始菜单”——“Microsoft SQL Server 2008”——“配置工具”——“Microsoft SQL Server 2008”选择“SQL server网络配置”将其中的“Named Pipes状态”“TCPIP状态”全部修改为“已启用”
如图：
![](http://blog.itpub.net/attachment/201403/13/24041331_1394710812Vrgv.gif)
  完成这些操作以后，你的问题就解决了。
