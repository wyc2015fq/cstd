## 方法/步骤
- 
运行CMD，telnet 127.0.0.1 1433，得到结果是连接失败。
关闭防火墙再测试，如果关了正常了说明是防火墙的问题，这个就不多说了。如果关了防火墙依然连接失败请继续往下看。
![SQL 数据库本地127.0.0.1无法登录](https://imgsa.baidu.com/exp/w=500/sign=75c5428a46c2d562f208d0edd71390f3/810a19d8bc3eb135ff075b51af1ea8d3fc1f441c.jpg)
- 
开始->程序，找到SQL Server 配置管理器
![SQL 数据库本地127.0.0.1无法登录](https://imgsa.baidu.com/exp/w=500/sign=fc71ee20efcd7b89e96c3a833f254291/b21bb051f81986181e0c016e43ed2e738ad4e695.jpg)
- 
在配置管理器中找到SQL Server网络配置下的MSSQLSERVER的协议
![SQL 数据库本地127.0.0.1无法登录](https://imgsa.baidu.com/exp/w=500/sign=2d5277c29e2bd40742c7d3fd4b889e9c/728da9773912b31b942724c38f18367adbb4e195.jpg)
- 
看右侧Named Pipes、TCP/IP、VIA是否是禁用的。禁用的话，双击选择启用
![SQL 数据库本地127.0.0.1无法登录](https://imgsa.baidu.com/exp/w=500/sign=51c97409f41f4134e037057e151d95c1/80cb39dbb6fd526627c1d5fea218972bd5073676.jpg)
- 
将Named Pipes、TCP/IP、VIA这状态全设置成已启用
![SQL 数据库本地127.0.0.1无法登录](https://imgsa.baidu.com/exp/w=500/sign=3ddab6f8464a20a4311e3cc7a0509847/342ac65c10385343853e69cb9a13b07ecb808863.jpg)
- 
回到SQL Server服务分页，重起SQL数据库
![SQL 数据库本地127.0.0.1无法登录](https://imgsa.baidu.com/exp/w=500/sign=2fda7a5cc0177f3e1034fc0d40ce3bb9/d50735fae6cd7b896043d269062442a7d9330e29.jpg)
- 
然后用SSMS以ip为127.0.0.1测试连接就能成功了
