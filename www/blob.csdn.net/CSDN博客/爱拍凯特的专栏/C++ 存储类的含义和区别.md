# C++ 存储类的含义和区别 - 爱拍凯特的专栏 - CSDN博客
2017年01月10日 21:43:40[爱拍凯特](https://me.csdn.net/apacat)阅读数：1838
C++中有五种类型的存储类。存储类是管理对象的生存期、链接、内存位置的类型说明符。这些说明符放在需要修饰的变量类型前。
auto
register
static
extern
mutable
1、说明符的作用是用来定义
变量/函数 的范围（可见性）和生命周期。
2、以上五类可以分成两种：自动存储类和静态存储类。自动存储类包括auto、register，其他的是静态存储类。
3、自动存储类修饰的变量在进入代码块之前生成，在进入代码块内存活，在出代码块之后废除。
4、只有变量可以作为自动存储类，且局部变量和参数变量默认是auto类型，auto修饰的变量存储在RAM，register类型存储在寄存器中，现在一般不用register，因为目前的编译器都是优化过的，编译器知道什么变量常用，就自动把它存储在寄存器中。
5、extern和static都是用来声明静态存储类变量和函数。这种变量从程序开始执行就存在，直到结束。对于变量来说，程序开始执行时就会给其修饰的变量分配内存空间，对于函数来说，程序开始执行时就存在函数名；但是，函数的作用域并不一定是整个程序。
6、静态存储类有两种标识符：外部标识符(如全局变量和函数名)与存储类说明符static中声明的局部变量。全局变量和函数名默认为存储类说明符extern，使用 static 修饰局部变量可以在函数调用之间保持局部变量的值。static
 修饰符也可以应用于全局变量。当 static 修饰全局变量时，会使变量的作用域限制在声明它的文件内。
7、mutable存储类只能用于类的数据成员，不能用于普通变量。具有mutable性质的类的数据成员打破了类对象的const限定，允许修改类的mutable的数据成员，即便类的其它成员仍然是const只读属性。
