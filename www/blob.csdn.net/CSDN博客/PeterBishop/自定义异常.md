这节课来学习自定义异常，非常非常简单，几分钟讲完。



自定义异常步骤:
- 新建一个类，类名建议以Exception结尾
- 如果你希望自定义的这个异常类是运行时异常，就让这个类继承RuntimeException，如果希望它是编译时异常，就直接继承Exception
- 重写构造器，构造器里直接调用父类构造器即可



为什么要自定义异常?

         很简单，换个名字而已，做到见名知义，比如如果文件操作出异常了就报IOException，数据库操作出错了就报SQLException，文件找不到就报FileNotFoundException，其实就是想换个名字而已。不信就来看看jdk提供的几个异常类吧



jdk的IOException:

```
public

class IOException extends Exception {

    static final long serialVersionUID = 7818375828146090155L;



    public IOException() {

        super();

    }



    

    public IOException(String message) {

        super(message);

    }



    

    public IOException(String message, Throwable cause) {

        super(message, cause);

    }



    

    public IOException(Throwable cause) {

        super(cause);

    }

}
```



可以看到IOException就只是继承了Exception类然后重写了构造器而已，由此可见自定义异常其实就是换个名字，做到见名知义而已。
