1039 到底买不买 （20 分）

小红想买些珠子做一串自己喜欢的珠串。卖珠子的摊主有很多串五颜六色的珠串，但是不肯把任何一串拆散了卖。于是小红要你帮忙判断一下，某串珠子里是否包含了全部自己想要的珠子？如果是，那么告诉她有多少多余的珠子；如果不是，那么告诉她缺了多少珠子。

为方便起见，我们用[0-9]、[a-z]、[A-Z]范围内的字符来表示颜色。例如在图1中，第3串是小红想做的珠串；那么第1串可以买，因为包含了全部她想要的珠子，还多了8颗不需要的珠子；第2串不能买，因为没有黑色珠子，并且少了一颗红色的珠子。

### 输入格式：

每个输入包含 1 个测试用例。每个测试用例分别在 2 行中先后给出摊主的珠串和小红想做的珠串，两串都不超过 1000 个珠子。

### 输出格式：

如果可以买，则在一行中输出 `Yes` 以及有多少多余的珠子；如果不可以买，则在一行中输出 `No` 以及缺了多少珠子。其间以 1 个空格分隔。

### 输入样例 1：

```
ppRYYGrrYBR2258
YrR8RrY
```

### 输出样例 1：

```
Yes 8
```

### 输入样例 2：

```
ppRYYGrrYB225
YrR8RrY
```

### 输出样例 2：

```
No 2
```


**思路：**这道题开始的思路是用两个数组存储，然后逐个比对，相等则继续下一个比对，不等则记录个数。如果下面字符串的长度加上记录的个数等于上面字符串的长度，说明是Yes。否则，是No。这种思路不好，无论是时间开销/空间开销，还是算法复杂度。

**在网上看到一个比较好的思路：**

> 
很简单且直观的方法：

还是用简单暴力的字符记录方法：使用int[128]数组记录每种字符（颜色）的数量，直接将字符的值作为索引。

（更新）只用一个数组来记录。第一行记录的时候增加计数，第二行记录的时候减少计数。那么正数表示这种颜色足够，负数表示这种颜色不足。

将正数和负数**分别累加**。如果缺少的数量累计值为0，说明足够——可以买，否则缺少的值表示缺少了多少珠子。

2018.12.04 这种统计字符个数的题目用C++的string类有时候反而不如int a[128]={0}方便。

**代码：**

```cpp
//PAT1039V1
#include <stdio.h>
int main(){
	char c;
	int record[128]={0},i;
	while((c=getchar())!='\n')	record[(int)c]++;
	while((c=getchar())!='\n')	record[(int)c]--;
	int more=0,less=0;
	for(i=0;i<128;i++){
		if(record[i]>0) more+=record[i];
		if(record[i]<0) less-=record[i];
	}
	
	if(less) printf("No %d",less);
	else	 printf("Yes %d",more);
	return 0;
}
```

这种键值的思想确实非常好用，时间空间和算法复杂度都很小。
