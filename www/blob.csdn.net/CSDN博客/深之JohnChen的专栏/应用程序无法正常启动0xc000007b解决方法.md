0xc000007b——应用程序无法正常启动

原因如下：
1、操作系统system32或SysWOW64中与vc相关dll与vs开发工具中的dll文件不一致。
2、缺乏dll。使用depends工具查看程序依赖那些dll。

用depends工具查询缺少哪些dll或dll文件冲突。

![](https://img-blog.csdn.net/20180521154044758)

红色：有两种含义。一个是没有找到这个文件，下面会有这个提示Could not find the section that owns the import directory.一个是有此文件，但dll版本不一致。

解决方法：

优先使用vs开发工具的动态库文件，也可以使用C:\Program Files (x86)\Microsoft Visual Studio 14.0\VC\redist\1033目录中拷贝运行库安装包。
1、C:\Program Files (x86)\Microsoft Visual Studio 14.0\VC\redist中x86或x64目录的Microsoft.VC140.CRT和Microsoft.VC140.MFC目录文件拷贝出来。（运行程序测试，是否可以运行，无法运行的话就执行下面的第二步。）

![](https://img-blog.csdn.net/20180521154544829)
2、C:\Program Files (x86)\Microsoft Visual Studio 14.0\Common7\IDE\Remote Debugger中x86或x64目录中dll文件拷贝出来。

![](https://img-blog.csdn.net/20180521154607555)
注意：C:\Program Files (x86)\Microsoft Visual Studio 14.0为vs2015安装目录
